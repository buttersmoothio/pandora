# Storage

Storage backends persist conversation history so Pandora remembers previous messages.

## Quick start

**SQLite (recommended)** — Conversations survive restarts:

```jsonc
"storage": {
  "type": "sqlite",
  "path": "data/pandora.db"
}
```

**Memory** — For testing (history lost on restart):

```jsonc
"storage": {
  "type": "memory"
}
```

---

## SQLite

File-based database. The default and recommended option.

```jsonc
"storage": {
  "type": "sqlite",
  "path": "data/pandora.db"
}
```

| Field | Default | Description |
|-------|---------|-------------|
| `type` | `sqlite` | Storage backend |
| `path` | `data/pandora.db` | Database file location |

### Features

- Conversations persist across restarts
- WAL mode for better performance
- Single file, easy to backup

### Backup

```bash
cp data/pandora.db data/pandora.db.backup
```

### Clear history

```bash
# Clear a specific conversation
sqlite3 data/pandora.db "DELETE FROM messages WHERE conversation_id = 'your-id';"

# Clear everything
rm data/pandora.db
```

---

## Memory

In-memory storage. All history is lost when Pandora stops.

```jsonc
"storage": {
  "type": "memory"
}
```

### When to use

- Development and testing
- Demos (clean slate each time)
- When you don't want persistent history

## Full example

```jsonc
{
  "ai": {
    "gateway": { "apiKey": "your-key" },
    "agents": {
      "operator": { "model": "google/gemini-3-flash" }
    }
  },
  "storage": {
    "type": "sqlite",
    "path": "data/pandora.db"
  },
  "channels": {
    "telegram": {
      "token": "bot-token",
      "ownerId": "user-id"
    }
  }
}
```
