# Security

## SES Lockdown

Pandora enables [SES (Secure EcmaScript)](https://github.com/endojs/endo/tree/master/packages/ses) lockdown at startup, before any other code runs. This freezes all JavaScript intrinsics (`Object.prototype`, `Array.prototype`, etc.), preventing prototype pollution attacks.

## Compartment Isolation

Tools run in isolated Compartments — separate global scopes with no access to host globals like `process`, `fetch`, or `require`. Each compartment receives only the capabilities declared in the tool's permissions. All endowments are `harden()`-ed before injection to prevent tampering.

Input and output cross the compartment boundary via JSON serialization, stripping any prototypes or compartment-internal references.

See the [Tools](/developer-guide/architecture/tools) architecture page for more on the permission model and sandbox design.

## SSRF Protection

Network endowments include server-side request forgery protection. Requests to private and loopback IP ranges are blocked regardless of the hostname allow-list. This prevents tools from probing internal infrastructure even if a public hostname resolves to a private address.

## Disclaimer

The sandbox and permission system builds on battle-tested primitives (SES lockdown, Compartment isolation, `harden()`), but Pandora's own integration code has not undergone a formal security audit. Treat it as best-effort defense-in-depth, not a guaranteed security boundary.
