# Building Custom Storage Providers

Every storage provider is a standalone package that exports a `createStorage` function.

## Core Types

All types are exported from `@pandora/core/storage`:

```ts
import type {
  AuthStore,
  Config,
  ConfigStore,
  StorageFactory,
  StorageResult,
} from '@pandora/core/storage'
```

### `StorageFactory`

The function signature your package must export:

```ts
type StorageFactory = (
  env: Record<string, string | undefined>,
  bindings?: unknown,
) => Promise<StorageResult>
```

- `env` — environment variables (from Hono's `env()` helper)
- `bindings` — Cloudflare Workers bindings (`c.env`), if applicable

### `StorageResult`

The object your factory must return:

```ts
interface StorageResult {
  /** Mastra storage for memory, workflows, etc. */
  mastra: MastraCompositeStore
  /** Pandora config storage */
  config: ConfigStore<Config>
  /** Pandora auth storage */
  auth: AuthStore
}
```

### `ConfigStore<T>`

The interface for config persistence:

```ts
interface ConfigStore<T = unknown> {
  /** Get the current config, or null if not set */
  get(): Promise<T | null>

  /** Save the config (full replacement) */
  set(config: T): Promise<void>

  /** Delete the config */
  delete(): Promise<void>

  /** Initialize tables/collections if needed (optional) */
  init?(): Promise<void>

  /** Close connections (optional cleanup) */
  close?(): Promise<void>
}
```

### `AuthStore`

The interface for auth persistence (credentials, sessions, refresh tokens):

```ts
interface AuthStore {
  init(): Promise<void>
  getCredential(): Promise<PasswordCredential | null>
  setCredential(credential: PasswordCredential): Promise<void>
  setCredentialIfNotExists(credential: PasswordCredential): Promise<boolean>
  createSession(session: Session): Promise<void>
  getSession(tokenHash: string): Promise<Session | null>
  deleteSession(tokenHash: string): Promise<void>
  deleteAllSessions(): Promise<void>
  listSessions(): Promise<Session[]>
  createRefreshToken(token: RefreshToken): Promise<void>
  getRefreshToken(tokenHash: string): Promise<RefreshToken | null>
  deleteRefreshToken(tokenHash: string): Promise<void>
  deleteAllRefreshTokens(): Promise<void>
  markRefreshTokenUsed(tokenHash: string): Promise<void>
  close?(): Promise<void>
}
```

## Built-in Helpers

Pandora provides reusable `ConfigStore` and `AuthStore` implementations that cover most databases.

<Tabs items={['SQL', 'Redis']}>
  <Tabs.Tab>
    For SQL databases (SQLite/LibSQL, PostgreSQL, MSSQL). You provide an `execute` function wrapping your driver:

    ```ts
    import { SQLConfigStore } from '@pandora/core/storage'

    const configStore = new SQLConfigStore<Config>(
      async (sql, params) => {
        const result = await myDriver.query(sql, params)
        return result.rows
      },
      'postgres' // dialect: 'sqlite' | 'postgres' | 'mssql'
    )
    ```

    The `execute` function receives raw SQL and an optional params array. It must return an array of row objects.

    Supported dialects handle differences in:
    - Table creation syntax
    - Parameterized query placeholders (`?`, `$1`, `@p1`)
    - Upsert strategy (`ON CONFLICT` vs `MERGE`)
    - JSON storage (`TEXT`, `JSONB`, `NVARCHAR(MAX)`)
  </Tabs.Tab>
  <Tabs.Tab>
    For Redis-like key-value stores. You provide an object with `get`, `set`, and `del` methods:

    ```ts
    import { RedisConfigStore } from '@pandora/core/storage'

    const configStore = new RedisConfigStore<Config>({
      get: (key) => myRedis.get(key),
      set: (key, value) => myRedis.set(key, value),
      del: (key) => myRedis.del(key),
    })
    ```

    Config is stored under the key `pandora:config` as a JSON string.
  </Tabs.Tab>
</Tabs>

### Auth Store Helpers

The same pattern applies for `AuthStore`. SQL-backed providers use `SQLAuthStore`, KV-backed providers use `RedisAuthStore`:

<Tabs items={['SQL', 'Redis']}>
  <Tabs.Tab>
    ```ts
    import { SQLAuthStore } from '@pandora/core/storage'

    const authStore = new SQLAuthStore(
      async (sql, params) => {
        const result = await myDriver.query(sql, params)
        return result.rows
      },
      'postgres' // dialect: 'sqlite' | 'postgres' | 'mssql'
    )
    ```

    Creates three tables: `pandora_auth_credentials`, `pandora_auth_sessions`, `pandora_auth_refresh_tokens`.
  </Tabs.Tab>
  <Tabs.Tab>
    ```ts
    import { RedisAuthStore } from '@pandora/core/storage'

    const authStore = new RedisAuthStore({
      get: (key) => myRedis.get(key),
      set: (key, value, opts) => myRedis.set(key, value, opts),
      del: (key) => myRedis.del(key),
      smembers: (key) => myRedis.smembers(key),
      sadd: (key, ...members) => myRedis.sadd(key, ...members),
      srem: (key, ...members) => myRedis.srem(key, ...members),
    })
    ```

    Uses key prefixes: `pandora:auth:credential`, `pandora:auth:session:*`, `pandora:auth:refresh:*`.
  </Tabs.Tab>
</Tabs>

If neither fits, implement `AuthStore` directly — see the MongoDB and DynamoDB providers for examples.

## Creating a Provider

Let's build a hypothetical `@pandora/storage-sqlite` provider as an example.

<Steps>

### Create the package

<FileTree>
  <FileTree.Folder name="packages/storage-sqlite" defaultOpen>
    <FileTree.File name="package.json" />
    <FileTree.Folder name="src" defaultOpen>
      <FileTree.File name="index.ts" />
    </FileTree.Folder>
    <FileTree.File name="tsconfig.json" />
  </FileTree.Folder>
</FileTree>

### Set up `package.json`

```json
{
  "name": "@pandora/storage-sqlite",
  "version": "0.0.1",
  "type": "module",
  "exports": {
    ".": "./src/index.ts"
  },
  "dependencies": {
    "better-sqlite3": "^11.0.0"
  },
  "peerDependencies": {
    "@pandora/core": "workspace:*"
  }
}
```

Key conventions:
- Package name must be `@pandora/storage-{name}` where `{name}` matches the `STORAGE_PROVIDER` value
- `@pandora/core` is a peer dependency
- Export `createStorage` from the package entry point

### Implement `createStorage`

```ts
// src/index.ts
import Database from 'better-sqlite3'
import type { Config, StorageFactory } from '@pandora/core/storage'
import { SQLAuthStore, SQLConfigStore } from '@pandora/core/storage'

export const createStorage: StorageFactory = async (env) => {
  const dbPath = env.DATABASE_PATH ?? 'pandora.db'
  const db = new Database(dbPath)

  const execute = async (sql: string, params?: unknown[]) => {
    const stmt = db.prepare(sql)
    return params ? stmt.all(...params) : stmt.all()
  }

  // Create the Mastra storage adapter
  // (use the appropriate @mastra/* package for your database)
  const mastra = createMastraAdapter(db)

  // Create config and auth stores using SQL helpers
  const config = new SQLConfigStore<Config>(execute, 'sqlite')
  const auth = new SQLAuthStore(execute, 'sqlite')

  return { mastra, config, auth }
}
```

### Test it

Set the environment variable and start Pandora:

```bash
STORAGE_PROVIDER=sqlite bun run dev
```

</Steps>

## Implementing Stores Directly

If the built-in helpers don't fit your database, implement `ConfigStore` and `AuthStore` directly. See the [MongoDB](/user-guide/storage/mongodb) and [DynamoDB](/user-guide/storage/dynamodb) providers for examples.

`ConfigStore` requirements:
- `get()` must return `null` (not throw) if no config exists yet
- `set()` performs a full replacement (upsert)
- `delete()` should not throw if the config doesn't exist
- `init()` is called automatically by Pandora after `createStorage` returns — use it for schema setup

`AuthStore` requirements:
- `init()` is called automatically — use it to create tables/collections
- `getCredential()` and `getSession()` must return `null` if not found
- `setCredentialIfNotExists()` must be atomic — return `true` if set, `false` if one already exists
