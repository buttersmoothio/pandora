# Build an Agent

Agent plugins add specialist agents that Pandora can route tasks to. Each plugin is automatically discovered via its `pandora.manifest.json`.

<Steps>

### Set up the package

An agent package follows this structure:

<FileTree>
  <FileTree.Folder name="research-agent" defaultOpen>
    <FileTree.File name="package.json" />
    <FileTree.File name="pandora.manifest.json" />
    <FileTree.Folder name="src" defaultOpen>
      <FileTree.File name="research.ts" />
    </FileTree.Folder>
    <FileTree.File name="tsconfig.json" />
  </FileTree.Folder>
</FileTree>

The `package.json` declares `@pandorakit/sdk` as a dev dependency (only type-only imports, if any, are used at runtime):

```json
{
  "name": "@pandorakit/research-agent",
  "type": "module",
  "exports": { ".": "./src/research.ts" },
  "devDependencies": {
    "@pandorakit/sdk": "*"
  }
}
```

### Write the manifest

The `pandora.manifest.json` declares the plugin's identity and agent capabilities:

```json
{
  "manifestVersion": 1,
  "id": "@pandorakit/research-agent",
  "name": "Research",
  "description": "Search the web and conduct in-depth research on any topic",
  "author": "Pandora",
  "version": "0.1.0",
  "license": "MIT",
  "pandora": ">=0.0.1",
  "provides": {
    "agents": [
      {
        "entry": "./src/research.ts",
        "useTools": [
          "@pandorakit/tavily-search:tavily_search",
          "@pandorakit/brave-search:brave_search",
          "@pandorakit/exa-search:exa_search",
          "@pandorakit/perplexity-search:perplexity_search"
        ],
        "modelTools": ["search"]
      }
    ]
  }
}
```

Key `provides.agents` fields:

| Field | Description |
|---|---|
| `entry` | Path to the file that exports the agent definition |
| `useTools` | Tool IDs from the global tool registry. Use fully-qualified IDs: `pluginId:toolId` |
| `modelTools` | Model-native capability keys. `"search"` maps to the configured provider's built-in search |

A plugin can provide multiple agents — `provides.agents` accepts an array of entries.

### Define the agent

Each agent entry point exports a named `agent` object — pure data with identity and instructions:

```ts
// src/research.ts
export const agent = {
  id: 'research',
  name: 'Deep Research',
  description: 'Thorough, multi-step research on complex topics',
  instructions: `You are a deep research specialist. Your job is to thoroughly
investigate complex questions by conducting multiple rounds of web searches,
cross-referencing sources, and synthesizing your findings.

When answering:
- Cite all sources with URLs
- Distinguish between well-established facts and claims with limited sourcing
- Note any conflicting information you found`,
}
```

No imports from `@pandorakit/sdk` needed. Agent entry points are pure data — the system reads identity and instructions from the exported object.

### Enable it

Pandora automatically discovers your plugin via its manifest. Users control it from the **Plugins** page:

- **Plugin level** — toggles the plugin on/off and stores settings from your `configFields`
- **Agent level** — within the plugin's settings dialog, users can configure per-agent model overrides

</Steps>

## How routing works

Pandora reads each agent's `description` and decides whether to handle a request directly or hand it off to a specialist. The routing decision is made by the LLM, not by rules — adding a new specialist is just installing a plugin, no routing configuration needed.

With zero agents registered, Pandora handles all requests directly — behavior is unchanged from a standard setup.

## Scoped tools (optional)

Agents can have their own tools that are scoped to them (not added to the global tool list). Define them as `Tool` objects and include them in the agent's `tools` array:

```ts
// src/my-agent.ts
export const agent = {
  id: 'my-agent',
  name: 'My Agent',
  description: 'An agent with scoped tools',
  instructions: 'Use your tools wisely.',
  tools: [
    {
      id: 'summarize',
      name: 'Summarize',
      description: 'Summarize a block of text',
      parameters: {
        type: 'object',
        properties: {
          text: { type: 'string', description: 'Text to summarize' },
        },
        required: ['text'],
      },
      execute: async (input: { text: string }) => {
        return { summary: input.text.slice(0, 500) }
      },
    },
  ],
}
```

## Reference

All types are exported from `@pandorakit/sdk/agents`:

```ts
import type { Agent } from '@pandorakit/sdk/agents'
```

### `Agent`

| Field | Type | Required | Description |
|---|---|---|---|
| `id` | `string` | Yes | Unique agent identifier |
| `name` | `string` | Yes | Human-readable display name |
| `description` | `string` | Yes | What the agent does (used for routing decisions) |
| `instructions` | `string` | Yes | System instructions for the agent |
| `tools` | `Tool[]` | No | Static scoped tools available to this agent |

Tool dependencies (`useTools`, `modelTools`) are declared in the manifest `provides.agents` entries, not in the agent code.
