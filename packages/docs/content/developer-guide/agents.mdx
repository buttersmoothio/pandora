# Build an Agent

Agent plugins add specialist agents that Pandora can route tasks to.

<Steps>

### Set up the package

An agent package follows this structure:

<FileTree>
  <FileTree.Folder name="pandora-agent-researcher" defaultOpen>
    <FileTree.File name="package.json" />
    <FileTree.Folder name="src" defaultOpen>
      <FileTree.File name="index.ts" />
      <FileTree.File name="researcher.ts" />
    </FileTree.Folder>
    <FileTree.File name="tsconfig.json" />
  </FileTree.Folder>
</FileTree>

The `package.json` declares `@pandora/core` as a peer dependency:

```json
{
  "name": "pandora-agent-researcher",
  "type": "module",
  "exports": { ".": "./src/index.ts" },
  "peerDependencies": {
    "@pandora/core": "*"
  }
}
```

### Define scoped tools (optional)

Agents can have their own tools that are scoped to them (not added to the global tool list). Define them with `defineTool()`:

```ts
// src/search.ts
import { z } from 'zod'
import { defineTool } from '@pandora/core/tools'

export const webSearch = defineTool({
  id: 'search',
  name: 'Web Search',
  description: 'Search the web for information',
  inputSchema: z.object({
    query: z.string().describe('Search query'),
  }),
  sandbox: 'host',
  execute: async ({ query }, { env }) => {
    // Your search implementation
    return { results: [] }
  },
})
```

### Define the agent

Use `defineAgent()` to create an agent with its identity, instructions, and tools:

```ts
// src/researcher.ts
import { defineAgent } from '@pandora/core/agents'
import { webSearch } from './search'

export const researcher = defineAgent({
  id: 'researcher',
  name: 'Researcher',
  description: 'Deep research on topics using web search and analysis',
  instructions: `You are a research specialist. When given a topic:
1. Search for authoritative sources
2. Cross-reference claims across multiple sources
3. Summarize findings with citations`,
  tools: [webSearch],
})
```

Multiple agents can share the same tool definition — just pass the same reference to each `defineAgent()` call.

### Export the plugin

The default export is an `AgentPlugin` descriptor:

```ts
// src/index.ts
import type { AgentPlugin } from '@pandora/core/agents'
import { researcher } from './researcher'

export default {
  id: 'agent-researcher',
  name: 'Researcher',
  schemaVersion: 1,
  agents: [researcher],
  envVars: [
    { name: 'SEARCH_API_KEY' },
  ],
} satisfies AgentPlugin
```

### Register it

Install your package in the Pandora instance and register it in `plugins.ts`:

```ts
import researcher from 'pandora-agent-researcher'
import { registerAgentPlugin } from './agents'

registerAgentPlugin(researcher)
```

### Enable it

Once registered, users control your plugin at two levels in **Settings → Agents**:

- **Plugin level** — toggles the plugin on/off and stores settings from your `configFields`. A plugin with required `configFields` is skipped until the user configures them.
- **Agent level** — toggles individual agents within the plugin and configures per-agent model overrides and scoped tool access.

</Steps>

## How routing works

Pandora reads each agent's `description` and decides whether to handle a request directly or hand it off to a specialist. The routing decision is made by the LLM, not by rules — adding a new specialist is just registering a plugin, no routing configuration needed.

With zero agents registered, Pandora handles all requests directly — behavior is unchanged from a standard setup.

## Reference

All types are exported from `@pandora/core/agents`:

```ts
import { defineAgent } from '@pandora/core/agents'

import type {
  AgentPlugin,
  ConfigFieldDescriptor,
  EnvVarDescriptor,
} from '@pandora/core/agents'
```

### `DefineAgentOptions`

| Field | Type | Required | Description |
|---|---|---|---|
| `id` | `string` | Yes | Unique agent identifier |
| `name` | `string` | Yes | Human-readable display name |
| `description` | `string` | Yes | What the agent does (used for routing decisions) |
| `instructions` | `string` | Yes | System instructions for the agent |
| `tools` | `ToolDefinition[]` | No | Scoped tools available to this agent |

### `AgentPlugin`

The descriptor your package default-exports:

| Field | Type | Required | Description |
|---|---|---|---|
| `id` | `string` | Yes | Plugin identifier, e.g. `'agent-researcher'` |
| `name` | `string` | Yes | Human-readable display name |
| `schemaVersion` | `number` | Yes | Must be `1` |
| `agents` | `AgentDefinition[]` | Yes | Agent definitions from `defineAgent()` |
| `envVars` | `EnvVarDescriptor[]` | No | Environment variable descriptors |
| `configFields` | `ConfigFieldDescriptor[]` | No | UI config field descriptors |
