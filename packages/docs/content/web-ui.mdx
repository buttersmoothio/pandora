# Web UI

Pandora's web interface is a Next.js app with real-time streaming, conversation management, and cross-channel visibility.

## Getting Started

1. Configure the web channel in `config.jsonc`:

```jsonc
"channels": {
  "web": {
    "port": 3000,
    "token": "pick-a-secret-token"
  }
}
```

2. Start everything:

```bash
bun run dev
```

3. Open `http://localhost:3001` and enter your token.

## Authentication

The web UI supports two ways to provide your token:

1. **URL parameter**: `http://localhost:3001?token=your-token` -- the token is validated and persisted in the URL.
2. **Environment variable**: Set `NEXT_PUBLIC_PANDORA_TOKEN` in your `.env.local` file for automatic login.

You can also configure a custom backend URL: `http://localhost:3001?backend=ws://custom-host:3000/ws`

## Features

### Conversation Sidebar

- Lists all conversations from all channels (Telegram, Web, etc.).
- Shows channel badge, message count, and last activity.
- Create new conversations or delete existing ones.
- Real-time updates when conversations change (via `watch-all`).

### Streaming Responses

- Text appears as it is generated.
- Tool calls and results shown in collapsible panels.
- Source citations displayed inline.
- Chain-of-thought/reasoning shown in a collapsible block with a "Thought for X seconds" timer.
- Token usage displayed per response.

### Subagent Thread Panel

- When the operator delegates to a subagent (coder, researcher, etc.), a side panel shows the subagent's work.
- Click on a tool call with a thread to open the panel.
- Shows the subagent's streaming text, tool calls, and reasoning.
- Status indicator: loading, streaming, done.

### Memory Panel

- When the operator recalls memories, they are shown in the message.
- Click on a memory item to open the detail panel.
- Shows full content, relevance score, category, and timestamps.
- Separate display for Facts vs Episodes.

### Cross-Channel Viewing

- Watch any conversation from any channel in real-time.
- See Telegram conversations update live in the web UI.
- Channel badges show where each message originated.

### Theme Toggle

- Light and dark mode (sun/moon toggle in the header).
- Persists via next-themes.

### Connection Status

- Shows connected/disconnected/reconnecting status.
- Auto-reconnects with exponential backoff (max 10 attempts, 1s--30s delays).
- Re-subscribes to watched conversations after reconnecting.

## Tech Stack

The web UI is built with:

- **Next.js 15** with App Router
- **shadcn/ui** for component primitives
- **AI Elements** for chat-specific components (message rendering, tool display, code blocks, etc.)
- **Streamdown** for markdown rendering with code highlighting, math, and mermaid support
- **Shiki** for syntax highlighting
- **motion/react** for animations
- **use-stick-to-bottom** for auto-scroll behavior

## Development

Run the web UI independently:

```bash
bun run --filter @pandora/web dev
```

The web UI connects to the backend at `ws://localhost:3000/ws` by default. The backend must be running for the web UI to function.
