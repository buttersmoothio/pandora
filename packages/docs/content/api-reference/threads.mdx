# Threads API

Endpoints for listing, reading, forking, and deleting conversation threads. See [API Conventions](/api-reference/conventions) for authentication and error format.

## GET /api/threads

List root threads, ordered by most recently updated. Supports pagination via `?page=0&perPage=50` query parameters.

<Tabs items={['API', 'SDK']}>
  <Tabs.Tab>
```bash
curl http://localhost:4111/api/threads \
  -H "Authorization: Bearer <token>"
```
  </Tabs.Tab>
  <Tabs.Tab>
```ts
const response = await client.threads.list()

console.log(response.data)
console.log(response.activeStreamIds)
```
  </Tabs.Tab>
</Tabs>


```json
{
  "data": [
    {
      "id": "abc123",
      "title": "Getting started with Pandora",
      "createdAt": "2025-01-01T00:00:00.000Z",
      "updatedAt": "2025-01-01T00:01:00.000Z",
      "activeThreadId": "fork-456",
      "threadIds": ["abc123", "fork-456"]
    }
  ],
  "total": 1,
  "page": 0,
  "perPage": 50,
  "hasMore": false,
  "activeStreamIds": ["abc123"]
}
```

| Field | Type | Description |
|---|---|---|
| `data` | `array` | Thread objects, ordered by `updatedAt` descending |
| `data[].id` | `string` | Root thread ID |
| `data[].title` | `string \| null` | Auto-generated title (set after first assistant response) |
| `data[].activeThreadId` | `string` | ID of the most recently updated thread in the family (root or any branch) |
| `data[].threadIds` | `string[]` | All thread IDs in the family (root + branches) |
| `total` | `number` | Total root thread count |
| `page` | `number` | Current page index |
| `perPage` | `number \| false` | Page size |
| `hasMore` | `boolean` | Whether more pages exist |
| `activeStreamIds` | `string[]` | Thread IDs currently streaming a response. Always `[]` in serverless mode. |

Only root threads appear in the list. Branches are accessible through the `threadIds` array or by fetching them directly with `GET /api/threads/:id`.

## GET /api/threads/:id

Get a single thread with its full message history and branching context.

<Tabs items={['API', 'SDK']}>
  <Tabs.Tab>
```bash
curl http://localhost:4111/api/threads/abc123 \
  -H "Authorization: Bearer <token>"
```
  </Tabs.Tab>
  <Tabs.Tab>
```ts
const detail = await client.threads.get('abc123')

console.log(detail.thread)
console.log(detail.messages)
console.log(detail.forks)
console.log(detail.forkInfo)
```
  </Tabs.Tab>
</Tabs>


```json
{
  "thread": {
    "id": "abc123",
    "title": "Getting started with Pandora",
    "createdAt": "2025-01-01T00:00:00.000Z",
    "updatedAt": "2025-01-01T00:01:00.000Z"
  },
  "messages": [
    {
      "id": "msg-1",
      "role": "user",
      "parts": [{ "type": "text", "text": "Hello" }]
    },
    {
      "id": "msg-2",
      "role": "assistant",
      "parts": [{ "type": "text", "text": "Hi there!" }]
    }
  ],
  "forks": {
    "msg-1": [{ "id": "fork-456", "title": "Alternative question" }]
  },
  "forkInfo": null
}
```

| Field | Type | Description |
|---|---|---|
| `thread` | `object` | Thread metadata |
| `messages` | `array` | Full message history. Each message has `id`, `role`, and `parts`. |
| `messages[].parts` | `Part[]` | Message content parts matching the [UIMessage](https://ai-sdk.dev/docs/reference/ai-sdk-core/ui-message) format. See [Message Part Types](/api-reference/message-parts). |
| `forks` | `Record<string, BranchRef[]>` | Child branches keyed by fork-point message ID. Each entry lists threads that branched at that message. |
| `forkInfo` | `ForkInfo \| null` | Present only when this thread is itself a branch. |

### ForkInfo

When a thread is a branch (created via fork), `forkInfo` describes its relationship to the parent:

| Field | Type | Description |
|---|---|---|
| `sourceThreadId` | `string` | The parent thread this was forked from |
| `forkPointIndex` | `number` | Index in the parent's message list where the fork occurred |
| `siblings` | `BranchRef[]` | Other branches forked from the same point in the parent |

A `BranchRef` is `{ id: string, title?: string }`.


| Status | Error | Cause |
|---|---|---|
| `404` | `Thread not found` | No thread with this ID exists |

## POST /api/threads/:id/fork

Create a new branch from an existing thread at a specific message. All messages **before** the fork point are cloned into the new thread. Returns `201`.

<Tabs items={['API', 'SDK']}>
  <Tabs.Tab>
```bash
curl -X POST http://localhost:4111/api/threads/abc123/fork \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"messageId": "msg-3"}'
```
  </Tabs.Tab>
  <Tabs.Tab>
```ts
const result = await client.threads.fork('abc123', 'msg-3')

console.log(result.thread.id)         // new branch ID
console.log(result.clonedMessageCount) // messages copied
```
  </Tabs.Tab>
</Tabs>

| Field | Type | Required | Description |
|---|---|---|---|
| `messageId` | `string` | Yes | ID of the message to fork at. Messages before this point are included in the branch. |


```json
{
  "thread": {
    "id": "fork-456",
    "createdAt": "2025-01-01T00:02:00.000Z",
    "updatedAt": "2025-01-01T00:02:00.000Z"
  },
  "clonedMessageCount": 4
}
```

After forking, send a [`POST /api/chat`](/api-reference/chat#post-apichat) with the new thread's ID to continue the conversation from the fork point.


| Status | Error | Cause |
|---|---|---|
| `400` | `messageId is required` | `messageId` is missing or not a string |
| `404` | `Message not found in thread` | The message ID does not exist in this thread |

## DELETE /api/threads/:id

Permanently delete a thread and all its messages.

<Tabs items={['API', 'SDK']}>
  <Tabs.Tab>
```bash
curl -X DELETE http://localhost:4111/api/threads/abc123 \
  -H "Authorization: Bearer <token>"
```
  </Tabs.Tab>
  <Tabs.Tab>
```ts
await client.threads.delete('abc123')
```
  </Tabs.Tab>
</Tabs>


```json
{ "id": "abc123" }
```


| Status | Error | Cause |
|---|---|---|
| `404` | `Thread not found` | No thread with this ID exists |
