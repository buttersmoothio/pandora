# Schedule API

Manage scheduled tasks programmatically.

## Endpoints

### List Tasks

Returns all scheduled tasks and the master enabled flag. Supports pagination via `?page=0&perPage=50` query parameters.

<Tabs items={['API', 'SDK']}>
  <Tabs.Tab>
```bash
curl http://localhost:4111/api/schedule \
  -H "Authorization: Bearer <token>"
```
  </Tabs.Tab>
  <Tabs.Tab>
```ts
const { enabled, data } = await client.schedule.list()
```
  </Tabs.Tab>
</Tabs>

```json
{
  "enabled": true,
  "data": [
    {
      "id": "a1b2c3d4-...",
      "name": "Daily summary",
      "cron": "0 8 * * *",
      "prompt": "Summarize my calendar for today",
      "enabled": true,
      "timezone": "America/New_York",
      "destination": "Web Inbox",
      "nextRun": "2026-03-05T08:00:00.000Z",
      "isRunning": false
    }
  ],
  "total": 1,
  "page": 0,
  "perPage": 50,
  "hasMore": false
}
```

### Create Task

Creates a new scheduled task. Returns `201` with the created task.

Exactly one of `cron` or `runAt` must be provided.

<Tabs items={['API', 'SDK']}>
  <Tabs.Tab>
```bash
curl -X POST http://localhost:4111/api/schedule \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Daily summary",
    "cron": "0 8 * * *",
    "prompt": "Summarize my calendar for today",
    "timezone": "America/New_York",
    "destination": "Web Inbox"
  }'
```
  </Tabs.Tab>
  <Tabs.Tab>
```ts
const task = await client.schedule.create({
  name: 'Daily summary',
  cron: '0 8 * * *',
  prompt: 'Summarize my calendar for today',
  timezone: 'America/New_York',
  destination: 'Web Inbox',
})
```
  </Tabs.Tab>
</Tabs>

### Update Task

Accepts a partial task object. Set optional fields to `null` to clear them (e.g., `"maxRuns": null`).

Setting `cron` clears `runAt` and vice versa.

<Tabs items={['API', 'SDK']}>
  <Tabs.Tab>
```bash
curl -X PATCH http://localhost:4111/api/schedule/a1b2c3d4 \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"enabled": false}'
```
  </Tabs.Tab>
  <Tabs.Tab>
```ts
const task = await client.schedule.update('a1b2c3d4', { enabled: false })
```
  </Tabs.Tab>
</Tabs>

## Heartbeat

### Get Heartbeat

Returns the heartbeat configuration and runtime status.

<Tabs items={['API', 'SDK']}>
  <Tabs.Tab>
```bash
curl http://localhost:4111/api/schedule/heartbeat \
  -H "Authorization: Bearer <token>"
```
  </Tabs.Tab>
  <Tabs.Tab>
```ts
const heartbeat = await client.schedule.heartbeat()
```
  </Tabs.Tab>
</Tabs>

```json
{
  "enabled": false,
  "cron": "*/30 * * * *",
  "tasks": [
    { "id": "a1b2c3d4-...", "description": "Check inbox for urgent emails", "enabled": true }
  ],
  "destination": "Telegram",
  "activeHours": { "start": "08:00", "end": "22:00" },
  "nextRun": "2026-03-05T08:30:00.000Z",
  "isRunning": false
}
```

### Update Heartbeat

Accepts a partial heartbeat config. Set optional fields to `null` to clear them (e.g., `"activeHours": null`).

<Tabs items={['API', 'SDK']}>
  <Tabs.Tab>
```bash
curl -X PATCH http://localhost:4111/api/schedule/heartbeat \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "enabled": true,
    "cron": "*/15 * * * *",
    "tasks": [
      { "id": "a1b2c3d4-...", "description": "Check inbox for urgent emails", "enabled": true }
    ]
  }'
```
  </Tabs.Tab>
  <Tabs.Tab>
```ts
const heartbeat = await client.schedule.updateHeartbeat({
  enabled: true,
  cron: '*/15 * * * *',
  tasks: [
    { id: 'a1b2c3d4-...', description: 'Check inbox for urgent emails', enabled: true },
  ],
})
```
  </Tabs.Tab>
</Tabs>

| Field | Type | Description |
|---|---|---|
| `enabled` | `boolean` | Whether heartbeat is active |
| `cron` | `string` | Interval as a cron expression (default: `*/30 * * * *`) |
| `tasks` | `array` | Checklist items. Each has `id` (UUID), `description` (string), and `enabled` (boolean). |
| `destination` | `string \| null` | Where to send alerts. One of the values from `/api/schedule/destinations`. |
| `activeHours` | `object \| null` | Optional. `{ "start": "HH:mm", "end": "HH:mm" }`. Restricts heartbeat to a time window. |
| `nextRun` | `string \| null` | Read-only. Next scheduled heartbeat time (ISO 8601). |
| `isRunning` | `boolean` | Read-only. Whether a heartbeat run is in progress. |

## Task Schema

| Field | Type | Required | Description |
|---|---|---|---|
| `id` | `string` | Auto | UUID, generated on creation |
| `name` | `string` | Yes | Human-readable name |
| `cron` | `string` | One of | Cron expression (mutually exclusive with `runAt`) |
| `runAt` | `string` | One of | ISO 8601 timestamp for one-time tasks |
| `prompt` | `string` | Yes | The prompt Pandora will process |
| `enabled` | `boolean` | No | Default `true` |
| `timezone` | `string` | No | IANA timezone for this task's schedule (e.g. `"America/New_York"`). Overrides the global timezone. |
| `maxRuns` | `number` | No | Max executions (recurring only). Task is removed when reached. |
| `destination` | `string` | No | Where to send results. One of the values from `/api/schedule/destinations`. |
| `nextRun` | `string \| null` | Read-only | Next scheduled execution time (ISO 8601) |
| `isRunning` | `boolean` | Read-only | Whether the task is currently executing |

## Task Execution

When a task fires:

1. Pandora processes the prompt in a dedicated thread
2. If a destination is set, results are delivered via the [Inbox](/api-reference/inbox) system
3. One-time tasks and tasks that reach their `maxRuns` limit are automatically removed
