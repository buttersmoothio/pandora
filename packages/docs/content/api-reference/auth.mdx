# Authentication API

Endpoint reference for authenticating requests and managing sessions. See [API Conventions](/api-reference/conventions) for the shared error format and status codes.

All auth routes are mounted at `/api/auth`. The setup, login, and refresh endpoints do not require authentication — all other endpoints require `Authorization: Bearer <token>`.

## Auth Status

Check whether auth has been set up and if the current request is authenticated:

<Tabs items={['API', 'SDK']}>
  <Tabs.Tab>
```bash
curl http://localhost:4111/
```
  </Tabs.Tab>
  <Tabs.Tab>
```ts
const health = await client.health()

console.log(health.auth.setup)         // true if password configured
console.log(health.auth.authenticated) // true if token is valid
```
  </Tabs.Tab>
</Tabs>

```json
{
  "name": "Pandora",
  "version": "0.1.0",
  "runtime": "node",
  "serverless": false,
  "auth": { "setup": true, "authenticated": false }
}
```

Use `auth.setup` to decide whether to show a setup or login flow.

## POST /api/auth/setup

Sets the initial password. Returns `201` with a token pair on success, `409` if already configured. Password must be at least 8 characters.

<Tabs items={['API', 'SDK']}>
  <Tabs.Tab>
```bash
curl -X POST http://localhost:4111/api/auth/setup \
  -H "Content-Type: application/json" \
  -d '{"password": "my-secure-password"}'
```
  </Tabs.Tab>
  <Tabs.Tab>
```ts
const tokens = await client.auth.setup('my-secure-password')

console.log(tokens.token)        // access token
console.log(tokens.refreshToken) // refresh token
```
  </Tabs.Tab>
</Tabs>

## POST /api/auth/login

## POST /api/auth/refresh

Rotates the refresh token and issues a new access token. The old refresh token is invalidated. Returns `401` if the token is expired, invalid, or reused.

<Tabs items={['API', 'SDK']}>
  <Tabs.Tab>
```bash
curl -X POST http://localhost:4111/api/auth/refresh \
  -H "Content-Type: application/json" \
  -d '{"refreshToken": "..."}'
```
  </Tabs.Tab>
  <Tabs.Tab>
```ts
const tokens = await client.auth.refresh('...')
```
  </Tabs.Tab>
</Tabs>

## POST /api/auth/change-password

Invalidates all sessions and refresh tokens, then returns a new token pair.

<Tabs items={['API', 'SDK']}>

```
  </Tabs.Tab>
  <Tabs.Tab>

```
  </Tabs.Tab>

## DELETE /api/auth/sessions/:id

Revoke a specific session. Returns `{ "id": "abc123", "loggedOut": false }` — `loggedOut` is `true` when you revoke your own session.

<Tabs items={['API', 'SDK']}>

  -H "Authorization: Bearer <token>"

## Response Formats

### Token Pair

All successful auth responses (`setup`, `login`, `refresh`, `change-password`) return the same shape:

```json
{
  "token": "...",
  "refreshToken": "...",
  "expiresAt": "2025-01-01T00:15:00.000Z",
  "refreshExpiresAt": "2025-01-08T00:00:00.000Z"
}
```

| Field | Type | Description |
|---|---|---|
| `token` | `string` | Access token — send as `Authorization: Bearer <token>` |
| `refreshToken` | `string` | Refresh token — use to obtain a new access token when the current one expires |
| `expiresAt` | `string` | Access token expiry (15 minutes from issue) |
| `refreshExpiresAt` | `string` | Refresh token expiry (7 days from issue) |

### Session List

```json
{
  "data": [
    {
      "id": "abc123...",
      "createdAt": "2025-01-01T00:00:00.000Z",
      "expiresAt": "2025-01-01T00:15:00.000Z",
      "userAgent": "Mozilla/5.0...",
      "ip": "192.168.1.1",
      "current": true
    }
  ]
}
```


| Code | Status | Meaning |
|---|---|---|
| `setup_required` | `403` | No password has been configured yet |
| `already_setup` | `409` | Password has already been configured |
| `invalid_credentials` | `401` | Wrong password |
| `unauthorized` | `401` | Missing or invalid access token |
| `refresh_token_required` | `400` | Refresh token not provided |

## Rate Limits

| Endpoint | Limit | Window |
|---|---|---|
| `/api/auth/setup` | 3 requests | 1 minute |
| `/api/auth/login` | 5 requests | 1 minute |
| `/api/auth/refresh` | 10 requests | 1 minute |
| `/api/auth/change-password` | 3 requests | 1 minute |

Rate-limited responses include `X-RateLimit-Limit`, `X-RateLimit-Remaining`, and `Retry-After` headers.
