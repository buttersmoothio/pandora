# Building Extensions

Pandora is designed to be extended. You can add:

- **Tools** — Give your AI new capabilities (weather, APIs, database queries)
- **Agents** — Create specialists (translator, writing assistant, data analyst)
- **Channels** — Connect to new platforms (Discord, Slack, CLI)
- **Storage** — Use different databases (Postgres, Redis, DynamoDB)

## How it works

Extensions are **auto-discovered**. Create a file in the right folder, and Pandora loads it automatically. No registration code, no index files to update.

```
packages/pandora/src/
├── tools/           # Your tools go here
│   ├── datetime.ts
│   └── weather.ts   ← Create this file, Pandora finds it
├── subagents/       # Your sub-agents go here
│   ├── coder.ts
│   └── translator.ts
├── channels/        # Your channels go here
│   ├── telegram/
│   └── discord/
└── store/           # Your storage backends go here
    ├── sqlite.ts
    └── postgres.ts
```

## File naming

| Pattern | Effect |
|---------|--------|
| `tools/*.ts` | Auto-discovered as tools |
| `subagents/*.ts` | Auto-discovered as sub-agents |
| `channels/*/index.ts` | Auto-discovered as channels |
| `store/*.ts` | Auto-discovered as storage backends |
| `_anything.ts` | Skipped (use for helpers) |

## Project structure

Pandora is a monorepo with two main packages:

```
packages/
├── core/        # Framework (don't modify)
│   └── src/
│       ├── agent.ts      # Agent runtime
│       ├── gateway.ts    # Message routing
│       └── registries/   # Extension registries
│
└── pandora/     # Your extensions (modify this!)
    └── src/
        ├── tools/        # Add tools here
        ├── subagents/    # Add agents here
        ├── channels/     # Add channels here
        └── store/        # Add storage here
```

**Key insight:** All customizations go in `packages/pandora/`. Import from `@pandora/core` but don't modify it.
