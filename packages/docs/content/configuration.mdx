# Configuration

Pandora uses a single file: `config.jsonc`. This page walks through the main sections. For an exhaustive field-by-field reference, see [Configuration Reference](/reference/configuration-reference).

## Getting started

Copy the example and edit:

```bash
cp config.example.jsonc config.jsonc
```

The config uses JSONC format — you can use comments (`//`) and trailing commas.

<Callout type="info">
For IDE autocompletion, add `"$schema": "./config.schema.jsonc"` at the top of your config file.
</Callout>

## Minimal config

The smallest recommended configuration:

```jsonc
{
  "$schema": "./config.schema.jsonc",
  "ai": {
    "gateway": { "apiKey": "YOUR_VERCEL_AI_GATEWAY_KEY" },
    "agents": {
      "operator": { "model": "google/gemini-3-flash" },
      "webSearchNative": { "model": "perplexity/sonar-pro" }
    }
  },
  "channels": {
    "telegram": {
      "token": "YOUR_BOT_TOKEN",
      "ownerId": "YOUR_USER_ID"
    }
  }
}
```

## AI Gateway

Access 50+ AI models with a single API key via the [Vercel AI Gateway](https://vercel.com/ai-gateway).

```jsonc
"ai": {
  "gateway": {
    "apiKey": "your-vercel-ai-gateway-key"
  }
}
```

### How model IDs work

All model references in config use the `provider/model` format — a single string that identifies both the AI provider and the specific model:

```
anthropic/claude-sonnet-4.5
openai/gpt-5.2
google/gemini-3-flash
perplexity/sonar-pro
openai/text-embedding-3-small
```

The provider prefix (before the `/`) tells the Vercel AI Gateway which backend to route to. The model suffix (after the `/`) selects the specific model from that provider. Browse the [full list of supported models](https://vercel.com/ai-gateway/models).

## Agents

Configure your AI assistants. Only `operator` is required.

```jsonc
"ai": {
  "agents": {
    "operator": { "model": "google/gemini-3-flash" },
    "coder": { "model": "anthropic/claude-sonnet-4.5" },
    "research": { "model": "openai/gpt-5.2" },
    "webSearchNative": { "model": "perplexity/sonar-pro" }
  }
}
```

See [Agents](/agents) for details on each agent's purpose.

## Tools

Capabilities you give to your agents.

```jsonc
"ai": {
  "tools": {
    "datetime": {},
    "tavilySearch": { "apiKey": "tvly-..." }
  }
}
```

See [Tools](/tools) for available tools and configuration.

## Storage

Where conversation history is saved. Defaults to SQLite if omitted.

```jsonc
"storage": {
  "type": "sqlite",
  "path": "data/pandora.db"
}
```

See [Storage](/storage) for details.

---

## Memory

Long-term recall across conversations. Optional — omit to disable.

```jsonc
"memory": {
  "type": "sqlite",
  "path": "data/memory.db",
  "embeddingModel": "openai/text-embedding-3-small"
}
```

See [Memory](/memory) for details.

---

## Channels

How users connect to Pandora. Each channel is enabled by being present in config.

```jsonc
"channels": {
  "telegram": {
    "token": "bot-token-from-botfather",
    "ownerId": "your-telegram-user-id"
  },
  "web": {
    "port": 3000,
    "token": "your-secret-token"
  }
}
```

See [Channels](/channels) for details.

---

## Log level

Controls how much detail Pandora prints to the console.

```jsonc
"logLevel": "normal"
```

| Value | Description |
|-------|-------------|
| `normal` | Metadata only: message flow, tool calls, durations (default) |
| `verbose` | Everything in normal, plus full model prompts, system instructions, and responses |
