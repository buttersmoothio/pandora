# Configuration

Pandora uses a single file: `config.jsonc`. This page covers every setting.

## Getting started

Copy the example and edit:

```bash
cp config.example.jsonc config.jsonc
```

The config uses JSONC format — you can use comments (`//`) and trailing commas.

For IDE autocompletion, add the schema reference:

```jsonc
{
  "$schema": "./config.schema.jsonc",
  // ... rest of config
}
```

## Minimal config

The smallest working configuration:

```jsonc
{
  "$schema": "./config.schema.jsonc",
  "ai": {
    "gateway": { "apiKey": "YOUR_VERCEL_AI_GATEWAY_KEY" },
    "agents": {
      "operator": { "model": "google/gemini-3-flash" }
    }
  },
  "channels": {
    "telegram": {
      "token": "YOUR_BOT_TOKEN",
      "ownerId": "YOUR_USER_ID"
    }
  }
}
```

## AI Gateway

Access 50+ AI models with a single API key.

```jsonc
"ai": {
  "gateway": {
    "apiKey": "your-vercel-ai-gateway-key"
  }
}
```

Get a key at [vercel.com/ai-gateway](https://vercel.com/ai-gateway).

## Agents

Configure your AI assistants. Only `operator` is required.

```jsonc
"ai": {
  "agents": {
    "operator": { "model": "google/gemini-3-flash" },
    "coder": { "model": "anthropic/claude-sonnet-4.5" },
    "research": { "model": "openai/gpt-5.2" },
    "webSearchNative": { "model": "perplexity/sonar-pro" }
  }
}
```

| Agent | Purpose |
|-------|---------|
| `operator` | Main AI (required) |
| `coder` | Programming specialist |
| `research` | Information and explanations |
| `webSearchNative` | Live web search |
| `webSearchTool` | Web search with external API |

See [Agents](/agents) for details.

## Tools

Capabilities you give to your agents.

```jsonc
"ai": {
  "tools": {
    "datetime": {},
    "tavilySearch": { "apiKey": "tvly-..." }
  }
}
```

| Tool | Config | Description |
|------|--------|-------------|
| `datetime` | `{}` | Current date/time |
| `tavilySearch` | `{ apiKey }` | Web search via Tavily |
| `exaSearch` | `{ apiKey }` | Web search via Exa |
| `perplexitySearch` | `{ apiKey }` | Web search via Perplexity |

See [Tools](/tools) for details.

## Storage

Where conversation history is saved.

```jsonc
"storage": {
  "type": "sqlite",
  "path": "data/pandora.db"
}
```

| Type | Description |
|------|-------------|
| `sqlite` | Persistent (default) |
| `memory` | In-memory, lost on restart |

See [Storage](/storage) for details.

---

## Memory

Long-term recall across conversations.

```jsonc
"memory": {
  "type": "sqlite",
  "path": "data/memory.db",
  "embeddingModel": "openai/text-embedding-3-small"
}
```

| Field | Default | Description |
|-------|---------|-------------|
| `type` | `sqlite` | Memory backend |
| `path` | `data/memory.db` | Database file location |
| `embeddingModel` | `openai/text-embedding-3-small` | Model for semantic search |

Memory is optional. See [Memory](/memory) for details.

## Channels

How users connect to Pandora. Each channel is enabled by being present in config.

### Telegram

```jsonc
"channels": {
  "telegram": {
    "token": "bot-token-from-botfather",
    "ownerId": "your-telegram-user-id"
  }
}
```

| Field | Description |
|-------|-------------|
| `token` | Bot token from [@BotFather](https://t.me/BotFather) |
| `ownerId` | Your user ID from [@userinfobot](https://t.me/userinfobot) |

### Web

```jsonc
"channels": {
  "web": {
    "port": 3000,
    "token": "your-secret-token"
  }
}
```

| Field | Default | Description |
|-------|---------|-------------|
| `port` | `3000` | HTTP/WebSocket port |
| `token` | — | Auth token for clients |

See [Channels](/channels) for details.

## Log level

| Value | Description |
|-------|-------------|
| `normal` | Standard logging (default) |

## Troubleshooting

### "Config validation failed"

The error message tells you which field is wrong:

```
Config validation error:
  ai.gateway.apiKey: Required
```

Fix: Add the missing field.

### "Unknown store type"

Use `sqlite` or `memory`, or create a [custom storage backend](/custom-storage).
