# Configuration API

Manage Pandora's agent configuration programmatically via the REST API.

## Endpoints

### Get Current Config

```
GET /api/config
Authorization: Bearer <token>
```

Returns the fully resolved config object (defaults merged with stored values).

### Update Config

```
PATCH /api/config
Authorization: Bearer <token>
Content-Type: application/json

{
  "identity": { "name": "My Assistant" },
  "models": {
    "operator": {
      "provider": "openai",
      "model": "gpt-4o"
    }
  }
}
```

Accepts a partial config object. Values are deep-merged with the existing config, validated, and persisted to storage.

### Reset to Defaults

```
DELETE /api/config
Authorization: Bearer <token>
```

Deletes the stored config and returns the default values.

## Config Schema

| Section | Description | Fields |
|---|---|---|
| `identity` | Agent name | `name` |
| `personality` | Behavior and tone | `systemPrompt` |
| `models` | LLM configuration | `operator: { provider, model, temperature?, maxTokens? }` |
| `tools` | Tool management | Keyed by tool ID: `{ enabled, settings?, requireApproval? }` |

## Available Tools

```
GET /api/tools
Authorization: Bearer <token>
```

Returns all registered stdlib tools with their manifests merged with the current config state. Unlike `GET /api/config` (which only returns tools already in config), this endpoint discovers all available tools regardless of whether they are enabled.

```json
{
  "tools": [
    {
      "id": "current-time",
      "description": "Get the current date and time in ISO 8601 format",
      "permissions": { "time": true },
      "sandbox": "host",
      "annotations": {
        "readOnlyHint": true,
        "destructiveHint": false,
        "idempotentHint": true
      },
      "enabled": true
    }
  ]
}
```

Each tool object contains:

| Field | Type | Description |
|---|---|---|
| `id` | `string` | Unique tool identifier |
| `description` | `string` | What the tool does |
| `permissions` | `object` | Declared capabilities (`time`, `network`, `env`, `fs`, `random`) |
| `sandbox` | `string` | `'compartment'` or `'host'` |
| `annotations` | `object?` | MCP-compatible UI hints (`title`, `readOnlyHint`, `destructiveHint`, `idempotentHint`) |
| `enabled` | `boolean` | Whether the tool is active in the current config |
| `requireApproval` | `boolean?` | Whether user approval is required before execution |
| `settings` | `object?` | Tool-specific settings from config |

## Available Models

```
GET /api/models
Authorization: Bearer <token>
```

Returns all supported providers, their models, and whether the required API key is configured:

```json
{
  "providers": [
    {
      "id": "anthropic",
      "name": "Anthropic",
      "models": ["claude-sonnet-4-20250514", "..."],
      "configured": true,
      "docUrl": "https://docs.anthropic.com",
      "gateway": null,
      "envVars": ["ANTHROPIC_API_KEY"]
    }
  ]
}
```

Use this endpoint to build model selection UIs. Providers with `configured: false` are missing their API key.
