# Configuration Reference

Exhaustive reference for every field in `config.jsonc`. For a guided overview of how to set up your configuration, see the [Configuration guide](/configuration).

Pandora uses a single JSONC (JSON with Comments) file at the monorepo root: **`config.jsonc`**. The file is gitignored because it contains secrets. Copy `config.example.jsonc` to get started:

```bash
cp config.example.jsonc config.jsonc
```

Add the `$schema` key for IDE autocompletion:

```jsonc
{
  "$schema": "./config.schema.jsonc"
}
```

---

## Top-level structure

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `$schema` | `string` | No | — | Path to the JSON Schema file for IDE autocompletion. |
| `ai` | `object` | **Yes** | — | AI configuration: gateway credentials, tools, and agent definitions. |
| `channels` | `object` | No | `{}` | Channel configurations. A channel is enabled by being present. |
| `storage` | `object` | No | `{ type: "sqlite", path: "data/pandora.db" }` | Conversation storage backend. |
| `memory` | `object` | No | — | Long-term memory (episodic + semantic). Omit to disable. |
| `logLevel` | `string` | No | `"normal"` | Logging verbosity: `"normal"` or `"verbose"`. |

---

## `ai`

The `ai` object is the only required top-level key. It configures the AI Gateway, tools, and agents.

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `ai.gateway` | `object` | **Yes** | — | Vercel AI Gateway credentials. |
| `ai.tools` | `object` | No | `{}` | Tool configurations keyed by tool name. |
| `ai.agents` | `object` | **Yes** | — | Agent configurations. Must include `operator`. |

### `ai.gateway`

Credentials for the [Vercel AI Gateway](https://vercel.com/ai-gateway), which provides a unified API to 50+ models across providers (OpenAI, Anthropic, Google, Perplexity, Mistral, and more).

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `apiKey` | `string` | **Yes** | — | Vercel AI Gateway API key. Must be non-empty. |

```jsonc
"gateway": {
  "apiKey": "your-ai-gateway-api-key"
}
```

The API key is used internally by the providers system in `@pandora/core`. When Pandora needs a language model (for agents) it calls `createModel(modelId, apiKey)`, which creates a gateway instance via `createGateway()` from `@ai-sdk/gateway` and resolves the `provider/model` string to the correct backend. For embedding models (used by the memory system), `createEmbeddingModel(modelId, apiKey)` calls `gateway.textEmbeddingModel()` instead. Both functions are called at runtime -- a fresh gateway instance is created for each model request.

### `ai.agents`

Agent configurations keyed by agent name. The `operator` agent is required; all others are optional sub-agents that the operator can delegate to.

Each agent accepts the following fields:

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `model` | `string` | **Yes** | — | Gateway model ID in `provider/model` format (e.g. `anthropic/claude-sonnet-4.5`, `openai/gpt-4o`, `google/gemini-2.0-flash`). |
| `temperature` | `number` | No | — | Sampling temperature. Range: `0` (deterministic) to `2` (maximum creativity). |
| `maxOutputTokens` | `number` | No | — | Maximum tokens in the model response. Must be a positive integer. |
| `maxSteps` | `number` | No | `20` | Maximum tool loop steps before the agent stops. Must be a positive integer. |
| `toolChoice` | `string` | No | — | Tool calling strategy: `"auto"`, `"required"`, or `"none"`. |
| `searchBackend` | `string` | No | — | Search backend tool name. Only used by the `webSearchTool` agent. |

#### Known agents

**`operator`** (required)

The main orchestrator agent. Handles general conversation and delegates to sub-agents when appropriate. This is the only required agent.

```jsonc
"operator": {
  "model": "anthropic/claude-sonnet-4.5"
}
```

**`coder`**

Specialized agent for programming tasks: debugging, code review, writing code, and explaining concepts.

```jsonc
"coder": {
  "model": "anthropic/claude-sonnet-4.5"
}
```

**`research`**

Specialized agent for information gathering: answering factual questions, explaining concepts, and providing summaries.

```jsonc
"research": {
  "model": "openai/gpt-4o"
}
```

**`webSearchNative`**

Web search using models with built-in search capabilities. Faster and cheaper (no extra API cost), but limited to specific models:

- Perplexity Sonar models (`perplexity/sonar-pro`, `perplexity/sonar`) -- search is built-in, no extra tools needed.
- OpenAI search models (`openai/gpt-4o-mini-search-preview`) -- uses the `web_search` provider tool. Requires `@ai-sdk/openai`.
- Google Gemini models (`google/gemini-2.0-flash`) -- uses the `googleSearch` provider tool. Requires `@ai-sdk/google`.

```jsonc
"webSearchNative": {
  "model": "perplexity/sonar-pro"
}
```

**`webSearchTool`**

Web search using an external search API tool. Works with any model but requires a search tool configured in `ai.tools` and the `searchBackend` field pointing to it.

| Extra field | Type | Required | Description |
|-------------|------|----------|-------------|
| `searchBackend` | `string` | **Yes** | Name of the search tool to use (must be configured in `ai.tools`). |

```jsonc
"webSearchTool": {
  "model": "anthropic/claude-sonnet-4.5",
  "searchBackend": "tavilySearch"
}
```

#### Custom agents

The `agents` object uses a catchall schema, so you can add any sub-agent name beyond the built-in ones. Custom sub-agents are defined via `defineSubagent()` in `packages/pandora/src/subagents/`. See [Creating extensions](/extensions/creating) for details.

### `ai.tools`

Tool configurations keyed by tool name. Each value is a free-form object containing tool-specific settings (typically an API key). A tool must be registered (auto-discovered from `packages/pandora/src/tools/`) to be used; unknown tool names cause a validation error.

#### Known tools

**`datetime`**

Returns the current date and time. No configuration needed.

```jsonc
"datetime": {}
```

**`tavilySearch`**

[Tavily](https://tavily.com/) AI-powered web search with high-quality results.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `apiKey` | `string` | **Yes** | Tavily API key. Get one at [tavily.com](https://tavily.com/). |

```jsonc
"tavilySearch": {
  "apiKey": "tvly-..."
}
```

**`exaSearch`**

[Exa](https://exa.ai/) AI-powered web search with advanced filtering and content extraction.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `apiKey` | `string` | **Yes** | Exa API key. Get one at [dashboard.exa.ai](https://dashboard.exa.ai/api-keys). |

```jsonc
"exaSearch": {
  "apiKey": "exa-..."
}
```

**`perplexitySearch`**

[Perplexity Search API](https://www.perplexity.ai/) with real-time web search. This is the tool-based Search API, distinct from the Perplexity Sonar chat models used by `webSearchNative`.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `apiKey` | `string` | **Yes** | Perplexity API key. Get one at [perplexity.ai/account/api/keys](https://www.perplexity.ai/account/api/keys). |

```jsonc
"perplexitySearch": {
  "apiKey": "pplx-..."
}
```

> **Note:** Memory tools (`remember`, `recall`, `getMemory`, `forget`) are auto-injected when the `memory` section is configured. They do not appear in `ai.tools`.

## `channels`

Channel configurations keyed by channel name. A channel is **enabled** by being present in this object and **disabled** by being absent. Each channel has its own required fields.

The schema uses `passthrough()`, so channel-specific fields are allowed without modifying the core schema.

### `telegram`

Telegram bot channel using [grammy](https://grammy.dev/). Single-owner bot -- only messages from the configured owner are processed.

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `token` | `string` | **Yes** | — | Telegram bot token from [@BotFather](https://t.me/BotFather). |
| `ownerId` | `string` | **Yes** | — | Telegram user ID of the bot owner. Get it from [@userinfobot](https://t.me/userinfobot). |

```jsonc
"telegram": {
  "token": "123456:ABC-DEF...",
  "ownerId": "987654321"
}
```

### `web`

Web channel providing HTTP REST endpoints and WebSocket streaming. Powers the Next.js web UI.

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `token` | `string` | **Yes** | — | Auth token that clients must provide to connect (via Bearer header or WebSocket query param). |
| `port` | `number` | No | `3000` | HTTP/WebSocket server port. |

```jsonc
"web": {
  "port": 3000,
  "token": "your-secret-auth-token"
}
```

### Custom channels

Additional channels can be registered via `defineChannel()` in `packages/pandora/src/channels/`. Any key in the `channels` object will be matched against registered channel names at startup.

## `storage`

Conversation history storage backend. Defaults to SQLite if omitted entirely.

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `type` | `string` | No | `"sqlite"` | Storage backend type. Built-in options: `"sqlite"`, `"memory"`. |
| `path` | `string` | No | `"data/pandora.db"` | Path to the database file. Relative to the monorepo root. Only used by file-based backends like SQLite. |

The schema uses `passthrough()`, so custom storage backends can accept additional fields.

### Built-in backends

**`sqlite`** -- Persistent, file-based storage using `bun:sqlite` with WAL mode. Recommended for production.

```jsonc
"storage": {
  "type": "sqlite",
  "path": "data/pandora.db"
}
```

**`memory`** -- In-memory storage. Data is lost when the process exits. Useful for development and testing.

```jsonc
"storage": {
  "type": "memory"
}
```

---

## `memory`

Long-term memory configuration. Enables two memory systems:

- **Episodic memory** -- Automatic logging of interactions, managed by the Gateway. Episodes are linked to conversations and deleted when conversations are deleted.
- **Semantic memory** -- Agent-controlled storage of facts, preferences, and knowledge via the `remember`, `recall`, `getMemory`, and `forget` tools.

Uses hybrid search: vector similarity (embeddings) combined with BM25 keyword matching.

Omit this section entirely to disable memory.

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `type` | `string` | No | `"sqlite"` | Memory backend type. |
| `path` | `string` | No | `"data/memory.db"` | Path to the memory database file. Separate from the conversation store. Relative to the monorepo root. |
| `embeddingModel` | `string` | No | `"openai/text-embedding-3-small"` | Gateway model ID in `provider/model` format for generating vector embeddings. Resolved via `createEmbeddingModel()` using the `ai.gateway.apiKey`. |

The schema uses `passthrough()`, so custom memory providers can accept additional fields.

```jsonc
"memory": {
  "type": "sqlite",
  "path": "data/memory.db",
  "embeddingModel": "openai/text-embedding-3-small"
}
```

> The Gateway API key (`ai.gateway.apiKey`) is automatically injected into the memory provider at runtime for embedding generation. You do not need to specify it separately.

## `logLevel`

Controls logging verbosity.

| Value | Description |
|-------|-------------|
| `"normal"` | Default. Logs metadata: message flow, tool calls, durations. |
| `"verbose"` | Additionally logs full model prompts and responses. Useful for debugging. |

```jsonc
"logLevel": "verbose"
```

---

## Validation rules

After parsing, `validateConfig()` runs the following checks. Startup fails if any rule is violated.

1. **Operator agent must be configured.** The `ai.agents.operator` object must be present with at least a `model` field.
2. **AI Gateway API key is required.** The `ai.gateway.apiKey` must be a non-empty string.
3. **Each tool in `ai.tools` must be a known registered tool.** Every key in `ai.tools` is checked against the tool registry (auto-discovered from `packages/pandora/src/tools/`). Unknown tool names produce an error.
4. **Zod schema validation.** All fields are validated against their Zod schemas at parse time:
   - `temperature` must be between `0` and `2`.
   - `maxOutputTokens` and `maxSteps` must be positive integers.
   - `toolChoice` must be one of `"auto"`, `"required"`, or `"none"`.
   - `logLevel` must be one of `"normal"` or `"verbose"`.

## Complete example

A fully configured `config.jsonc` with all sections populated:

```jsonc
{
  // IDE autocompletion
  "$schema": "./config.schema.jsonc",

  "ai": {
    "gateway": {
      "apiKey": "your-ai-gateway-api-key"
    },

    "tools": {
      "datetime": {},
      "tavilySearch": {
        "apiKey": "tvly-your-tavily-api-key"
      },
      "exaSearch": {
        "apiKey": "exa-your-exa-api-key"
      },
      "perplexitySearch": {
        "apiKey": "pplx-your-perplexity-api-key"
      }
    },

    "agents": {
      // Required: main orchestrator
      "operator": {
        "model": "anthropic/claude-sonnet-4.5",
        "temperature": 0.7,
        "maxSteps": 20
      },

      // Optional sub-agents
      "coder": {
        "model": "anthropic/claude-sonnet-4.5",
        "temperature": 0.2
      },
      "research": {
        "model": "openai/gpt-4o",
        "maxOutputTokens": 4096
      },

      // Web search: pick ONE of these two approaches
      "webSearchNative": {
        "model": "perplexity/sonar-pro"
      },
      "webSearchTool": {
        "model": "anthropic/claude-sonnet-4.5",
        "searchBackend": "tavilySearch"
      }
    }
  },

  "channels": {
    "telegram": {
      "token": "123456789:ABCdefGHIjklMNOpqrsTUVwxyz",
      "ownerId": "987654321"
    },
    "web": {
      "port": 3000,
      "token": "your-secret-auth-token"
    }
  },

  "storage": {
    "type": "sqlite",
    "path": "data/pandora.db"
  },

  "memory": {
    "type": "sqlite",
    "path": "data/memory.db",
    "embeddingModel": "openai/text-embedding-3-small"
  },

  "logLevel": "normal"
}
```
