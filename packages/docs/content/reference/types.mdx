# Types

Core TypeScript types exported from `@pandora/core`. These types define the contracts between channels, the gateway, and the agent.

For streaming event types, see [Streaming & Events](/reference/streaming). For store types, see [Store](/reference/store). For memory types, see [Memory](/reference/memory-reference).

All types can be imported directly:

```ts
import type { Message, Channel, ChannelCapabilities, RequestContext } from "@pandora/core";
```

---

## Message Types

Types that represent messages flowing through the system, from channel input to stored conversation history.

### Message

Incoming message from a channel, passed through the gateway to the agent.

```ts
interface Message {
  /** Which channel this message came from */
  channelName: string;
  /** User identifier (channel-specific format) */
  userId: string;
  /** Conversation/chat identifier */
  conversationId: string;
  /** Text content of the message */
  content: string;
  /** Optional attachments (images, files, audio, video) */
  attachments?: Attachment[];
  /** Message ID to reply to (for quoting) */
  replyToMessageId?: number;
  /** Channel-specific metadata */
  metadata?: Record<string, unknown>;
}
```

| Field | Type | Description |
|---|---|---|
| `channelName` | `string` | Identifies which channel sent the message (e.g. `"telegram"`, `"web"`). |
| `userId` | `string` | User identifier in the channel's native format. |
| `conversationId` | `string` | Groups messages into a conversation. Format is channel-specific. |
| `content` | `string` | The text body of the message. |
| `attachments` | `Attachment[]` | Optional array of file/media attachments. |
| `replyToMessageId` | `number` | Optional ID of a message being replied to (for quoting). |
| `metadata` | `Record<string, unknown>` | Arbitrary channel-specific metadata. |

### Attachment

A file or media item attached to a message. At least one of `fileId`, `url`, or `data` should be present.

```ts
interface Attachment {
  type: "image" | "file" | "audio" | "video";
  /** File identifier (channel-specific, e.g. Telegram file_id) */
  fileId?: string;
  /** Direct URL to the file */
  url?: string;
  /** Raw file data */
  data?: Uint8Array;
  /** MIME type of the file */
  mimeType?: string;
  /** Original filename */
  filename?: string;
  /** File size in bytes */
  size?: number;
  /** Duration in seconds (for audio/video) */
  duration?: number;
  /** Caption text (for images/files with captions) */
  caption?: string;
}
```

| Field | Type | Description |
|---|---|---|
| `type` | `"image" \| "file" \| "audio" \| "video"` | The media type of the attachment. |
| `fileId` | `string` | Platform-specific file identifier (e.g. Telegram `file_id`). |
| `url` | `string` | Direct URL to access the file. |
| `data` | `Uint8Array` | Raw binary file data. |
| `mimeType` | `string` | MIME type (e.g. `"image/png"`, `"audio/ogg"`). |
| `filename` | `string` | Original filename. |
| `size` | `number` | File size in bytes. |
| `duration` | `number` | Duration in seconds. Only meaningful for `audio` and `video` types. |
| `caption` | `string` | Caption text accompanying the attachment. |

### MessageMeta

Metadata attached to stored messages. Used by the store to track message provenance.

```ts
interface MessageMeta {
  /** Which channel this message originated from */
  channelName?: string;
  /** User identifier (channel-specific format) */
  userId?: string;
  /** Unix epoch seconds when created */
  createdAt?: number;
}
```

### PandoraMessage

Alias for the AI SDK's `UIMessage` type. This is the stored message format, using parts-based storage for tool calls, reasoning, sources, and text.

```ts
// Re-exported from the `ai` package
type PandoraMessage = UIMessage;
```

### MemoryContextPart

A custom message part representing memory context recalled for a prompt.

```ts
interface MemoryContextPart {
  type: "memory-context";
  facts: Array<{ content: string; category?: string; score: number }>;
  episodes: Array<{ content: string; timestamp?: number; score: number }>;
}
```

### PandoraMessagePart

Union of all AI SDK message part types plus Pandora-specific parts.

```ts
type PandoraMessagePart = UIMessagePart | MemoryContextPart;
```

## Channel Types

Types that define channel capabilities and the channel interface.

### ChannelCapabilities

Declares what a channel supports. These are fixed characteristics defined in code, not runtime state.

```ts
interface ChannelCapabilities {
  /** Can send/receive images */
  supportsImages: boolean;
  /** Can send/receive files */
  supportsFiles: boolean;
  /** Supports rich text formatting (Markdown, HTML) */
  supportsRichText: boolean;
  /** Supports interactive buttons/keyboards */
  supportsButtons: boolean;
  /** Can stream responses token-by-token */
  supportsStreaming: boolean;
  /** Character limit per message (-1 for unlimited) */
  maxMessageLength: number;
  /** Can proactively send messages to users (for scheduled tasks) */
  supportsPush: boolean;
}
```

| Field | Type | Description |
|---|---|---|
| `supportsImages` | `boolean` | Whether the channel can send and receive images. |
| `supportsFiles` | `boolean` | Whether the channel can send and receive file attachments. |
| `supportsRichText` | `boolean` | Whether the channel supports rich text (Markdown, HTML). |
| `supportsButtons` | `boolean` | Whether the channel supports interactive buttons or inline keyboards. |
| `supportsStreaming` | `boolean` | Whether the channel can stream responses token-by-token. |
| `maxMessageLength` | `number` | Maximum characters per message. Use `-1` for unlimited. |
| `supportsPush` | `boolean` | Whether the channel can proactively send messages (for scheduled reminders). |

### Channel

Interface that all channel implementations must satisfy. Channels are registered via `defineChannel()` and discovered at startup.

```ts
interface Channel {
  /** Unique name for this channel */
  readonly name: string;
  /** Channel's capabilities (what it can do) */
  readonly capabilities: ChannelCapabilities;
  /** Start the channel (begin listening for messages) */
  start(): Promise<void>;
  /** Stop the channel gracefully */
  stop(): Promise<void>;
}
```

### ChannelPusher

Extension of Channel for channels that support proactive push notifications. Channels with `supportsPush: true` should implement this interface.

```ts
interface ChannelPusher extends Channel {
  /**
   * Send a proactive message to a user.
   * Used for scheduled reminders and notifications.
   *
   * @param userId - User identifier (channel-specific format)
   * @param content - Message content to send
   */
  push(userId: string, content: string): Promise<void>;
}
```

### MessageHandler

Handler function type for processing messages through the gateway. Used by non-streaming channels.

```ts
type MessageHandler = (
  message: Message,
  capabilities: ChannelCapabilities
) => Promise<string>;
```

### StreamingMessageHandler

Handler that yields text deltas as they arrive from the agent. Used by streaming-capable channels.

```ts
type StreamingMessageHandler = (
  message: Message,
  capabilities: ChannelCapabilities
) => AsyncGenerator<string, void>;
```

## Request Context

Per-request state available to tools and other code running within a gateway request. Set automatically by the Gateway via `AsyncLocalStorage`.

```ts
interface RequestContext {
  /** Current conversation ID */
  conversationId: string;
  /** Channel this request originated from */
  channelName: string;
  /** User ID this request originated from */
  userId: string;
}
```

| Field | Type | Description |
|---|---|---|
| `conversationId` | `string` | The conversation being processed |
| `channelName` | `string` | The channel the message originated from |
| `userId` | `string` | The user ID in the channel's native format |

## Context Management Types

Types for tracking context window usage, costs, and health. These are emitted via stream events when context management is enabled.

For event schemas, see [Streaming & Events](/reference/streaming#context-management-events).

### TokenUsage

Token counts from a single turn/step.

```ts
interface TokenUsage {
  inputTokens: number;
  outputTokens: number;
  reasoningTokens?: number;
  cachedInputTokens?: number;
  totalTokens: number;
}
```

### TokenCosts

USD cost breakdown for a conversation.

```ts
interface TokenCosts {
  inputUSD: number;
  outputUSD: number;
  reasoningUSD: number;
  cacheReadUSD: number;
  cacheWriteUSD: number;
  totalUSD: number;
}
```

### ContextHealth

Current context window health status.

```ts
interface ContextHealth {
  /** Tokens currently in context */
  usedTokens: number;
  /** Tokens available for new input */
  remainingTokens: number;
  /** Usage as percentage (0-100) */
  percentUsed: number;
  /** Context is within safe limits */
  isHealthy: boolean;
  /** Compaction recommended */
  shouldCompact: boolean;
  /** Tokens to remove if compacting */
  tokensToRemove: number;
}
```

### ContextLimits

Model context limits from TokenLens metadata.

```ts
interface ContextLimits {
  /** Max input tokens */
  input: number;
  /** Max output tokens */
  output: number;
  /** Max total context (input + output) */
  total: number;
}
```

### ContextState

Complete context state for an agent, emitted via `context-state` events.

```ts
interface ContextState {
  /** Model identifier (e.g., "anthropic/claude-sonnet-4.5") */
  modelId: string;
  /** Context limits */
  limits: ContextLimits;
  /** Current health status */
  health: ContextHealth;
  /** Accumulated costs */
  costs: TokenCosts;
  /** Usage from last completed turn */
  lastTurn?: TokenUsage;
}
```

### ConversationStats

Aggregated stats across operator and subagents, emitted via `conversation-stats` events.

```ts
interface ConversationStats {
  /** Operator context state */
  operator: ContextState;
  /** Subagent states by threadId */
  subagents: Record<string, ContextState>;
  /** Total cost across all agents */
  totalCostUSD: number;
  /** Total input tokens (last turn, all agents) */
  totalInputTokens: number;
  /** Total output tokens (last turn, all agents) */
  totalOutputTokens: number;
}
```

## Scheduled Task Types

Types for the scheduler system. See [Store](/reference/store#scheduled-task-methods) for the store methods that use these types.

### ScheduledTask

A scheduled task record stored in the database.

```ts
interface ScheduledTask {
  id: string;
  conversationId: string;
  channelName: string;
  userId: string;

  /** Task scheduling type */
  type: "once" | "recurring";
  /** Task category for display and behavior */
  taskType: "reminder" | "followup" | "custom";
  /** Human-readable description of what to do */
  description: string;
  /** Optional additional context (JSON blob) */
  context?: Record<string, unknown>;

  /** For one-time tasks: when to run (Unix epoch seconds) */
  runAt?: number;
  /** For recurring tasks: cron expression */
  cronExpression?: string;
  /** Timezone for cron interpretation */
  timezone: string;

  /** Current task status */
  status: "pending" | "running" | "completed" | "cancelled" | "failed";
  /** Last execution time (Unix epoch seconds) */
  lastRunAt?: number;
  /** Number of times this task has run */
  runCount: number;
  /** Maximum number of runs (recurring only, null = unlimited) */
  maxRuns?: number;

  /** Last error message if failed */
  lastError?: string;
  /** Number of consecutive failures */
  failureCount: number;
  /** Maximum failures before marking as failed */
  maxFailures: number;

  /** Unix epoch seconds */
  createdAt: number;
  /** Unix epoch seconds */
  updatedAt: number;
}
```

### CreateScheduledTaskInput

Input for creating a new scheduled task.

```ts
interface CreateScheduledTaskInput {
  conversationId: string;
  channelName: string;
  userId: string;

  type: "once" | "recurring";
  taskType: "reminder" | "followup" | "custom";
  description: string;
  context?: Record<string, unknown>;

  /** For one-time tasks: when to run (Unix epoch seconds) */
  runAt?: number;
  /** For recurring tasks: cron expression */
  cronExpression?: string;
  timezone?: string;
  maxRuns?: number;
  maxFailures?: number;
}
```

### UpdateScheduledTaskInput

Partial update for a scheduled task.

```ts
interface UpdateScheduledTaskInput {
  status?: "pending" | "running" | "completed" | "cancelled" | "failed";
  lastRunAt?: number;
  runCount?: number;
  lastError?: string;
  failureCount?: number;
}
```
