# Types & Interfaces

Complete reference for all TypeScript types and interfaces exported from `@pandora/core`. These types define the contracts between channels, the gateway, the agent, storage backends, and memory providers.

All types can be imported directly:

```ts
import type { Message, Channel, StreamEvent, IMessageStore } from "@pandora/core";
```

---

## Message Types

Types that represent messages flowing through the system, from channel input to stored conversation history.

### Message

Incoming message from a channel, passed through the gateway to the agent.

```ts
interface Message {
  /** Which channel this message came from */
  channelName: string;
  /** User identifier (channel-specific format) */
  userId: string;
  /** Conversation/chat identifier */
  conversationId: string;
  /** Text content of the message */
  content: string;
  /** Optional attachments (images, files, audio, video) */
  attachments?: Attachment[];
  /** Message ID to reply to (for quoting) */
  replyToMessageId?: number;
  /** Channel-specific metadata */
  metadata?: Record<string, unknown>;
}
```

| Field | Type | Description |
|---|---|---|
| `channelName` | `string` | Identifies which channel sent the message (e.g. `"telegram"`, `"web"`). |
| `userId` | `string` | User identifier in the channel's native format. |
| `conversationId` | `string` | Groups messages into a conversation. Format is channel-specific. |
| `content` | `string` | The text body of the message. |
| `attachments` | `Attachment[]` | Optional array of file/media attachments. |
| `replyToMessageId` | `number` | Optional ID of a message being replied to (for quoting). |
| `metadata` | `Record<string, unknown>` | Arbitrary channel-specific metadata. |

### Attachment

A file or media item attached to a message. At least one of `fileId`, `url`, or `data` should be present.

```ts
interface Attachment {
  type: "image" | "file" | "audio" | "video";
  /** File identifier (channel-specific, e.g. Telegram file_id) */
  fileId?: string;
  /** Direct URL to the file */
  url?: string;
  /** Raw file data */
  data?: Uint8Array;
  /** MIME type of the file */
  mimeType?: string;
  /** Original filename */
  filename?: string;
  /** File size in bytes */
  size?: number;
  /** Duration in seconds (for audio/video) */
  duration?: number;
  /** Caption text (for images/files with captions) */
  caption?: string;
}
```

| Field | Type | Description |
|---|---|---|
| `type` | `"image" \| "file" \| "audio" \| "video"` | The media type of the attachment. |
| `fileId` | `string` | Platform-specific file identifier (e.g. Telegram `file_id`). |
| `url` | `string` | Direct URL to access the file. |
| `data` | `Uint8Array` | Raw binary file data. |
| `mimeType` | `string` | MIME type (e.g. `"image/png"`, `"audio/ogg"`). |
| `filename` | `string` | Original filename. |
| `size` | `number` | File size in bytes. |
| `duration` | `number` | Duration in seconds. Only meaningful for `audio` and `video` types. |
| `caption` | `string` | Caption text accompanying the attachment. |

### MessageMeta

Metadata attached to stored messages. Used by the store to track message provenance.

```ts
interface MessageMeta {
  /** Which channel this message originated from */
  channelName?: string;
  /** User identifier (channel-specific format) */
  userId?: string;
  /** Unix epoch seconds when created */
  createdAt?: number;
}
```

### PandoraMessage

Alias for the AI SDK's `UIMessage` type. This is the stored message format, using parts-based storage for tool calls, reasoning, sources, and text.

```ts
// Re-exported from the `ai` package
type PandoraMessage = UIMessage;
```

### MemoryContextPart

A custom message part representing memory context recalled for a prompt.

```ts
interface MemoryContextPart {
  type: "memory-context";
  facts: Array<{ content: string; category?: string; score: number }>;
  episodes: Array<{ content: string; timestamp?: number; score: number }>;
}
```

### PandoraMessagePart

Union of all AI SDK message part types plus Pandora-specific parts.

```ts
type PandoraMessagePart = UIMessagePart | MemoryContextPart;
```

## Channel Types

Types that define channel capabilities and the channel interface.

### ChannelCapabilities

Declares what a channel supports. These are fixed characteristics defined in code, not runtime state.

```ts
interface ChannelCapabilities {
  /** Can send/receive images */
  supportsImages: boolean;
  /** Can send/receive files */
  supportsFiles: boolean;
  /** Supports rich text formatting (Markdown, HTML) */
  supportsRichText: boolean;
  /** Supports interactive buttons/keyboards */
  supportsButtons: boolean;
  /** Can stream responses token-by-token */
  supportsStreaming: boolean;
  /** Character limit per message (-1 for unlimited) */
  maxMessageLength: number;
}
```

| Field | Type | Description |
|---|---|---|
| `supportsImages` | `boolean` | Whether the channel can send and receive images. |
| `supportsFiles` | `boolean` | Whether the channel can send and receive file attachments. |
| `supportsRichText` | `boolean` | Whether the channel supports rich text (Markdown, HTML). |
| `supportsButtons` | `boolean` | Whether the channel supports interactive buttons or inline keyboards. |
| `supportsStreaming` | `boolean` | Whether the channel can stream responses token-by-token. |
| `maxMessageLength` | `number` | Maximum characters per message. Use `-1` for unlimited. |

### Channel

Interface that all channel implementations must satisfy. Channels are registered via `defineChannel()` and discovered at startup.

```ts
interface Channel {
  /** Unique name for this channel */
  readonly name: string;
  /** Channel's capabilities (what it can do) */
  readonly capabilities: ChannelCapabilities;
  /** Start the channel (begin listening for messages) */
  start(): Promise<void>;
  /** Stop the channel gracefully */
  stop(): Promise<void>;
}
```

### MessageHandler

Handler function type for processing messages through the gateway. Used by non-streaming channels.

```ts
type MessageHandler = (
  message: Message,
  capabilities: ChannelCapabilities
) => Promise<string>;
```

### StreamingMessageHandler

Handler that yields text deltas as they arrive from the agent. Used by streaming-capable channels.

```ts
type StreamingMessageHandler = (
  message: Message,
  capabilities: ChannelCapabilities
) => AsyncGenerator<string, void>;
```

## Stream Events

Events emitted during streaming for real-time visibility into agent processing. Delivered via the `onEvent` callback alongside the text stream.

### StreamEvent

A discriminated union of all events that can be emitted during a streaming response. Each variant has a `type` field and an optional `threadId` to scope it to a subagent thread (`null`/`undefined` = operator, `string` = subagent thread).

```ts
type StreamEvent =
  | { type: "text-delta"; text: string; threadId?: string }
  | { type: "tool-call"; toolCallId: string; toolName: string; args: unknown; threadId?: string }
  | { type: "tool-result"; toolCallId: string; toolName: string; result: unknown; threadId?: string }
  | { type: "source-url"; sourceId: string; url: string; title?: string;
      providerMetadata?: Record<string, unknown>; threadId?: string }
  | { type: "source-document"; sourceId: string; mediaType: string; title: string;
      filename?: string; providerMetadata?: Record<string, unknown>; threadId?: string }
  | { type: "reasoning-delta"; text: string; threadId?: string }
  | { type: "step-start"; threadId?: string }
  | { type: "step-finish"; usage: { inputTokens?: number; outputTokens?: number;
      totalTokens?: number }; finishReason: string; threadId?: string }
  | { type: "file"; mediaType: string; url: string; filename?: string; threadId?: string }
  | { type: "memory-context";
      facts: Array<{ content: string; category?: string; score: number }>;
      episodes: Array<{ content: string; timestamp?: number; score: number }> }
  | { type: "subagent-start"; threadId: string; toolCallId: string; subagentName: string }
  | { type: "subagent-done"; threadId: string };
```

#### Event Variants

| Type | Key Fields | Description |
|---|---|---|
| `text-delta` | `text` | A chunk of generated text. Accumulate these to build the full response. |
| `tool-call` | `toolCallId`, `toolName`, `args` | The agent is invoking a tool. `args` contains the tool's input. |
| `tool-result` | `toolCallId`, `toolName`, `result` | A tool has returned its result. `result` is the tool's output. |
| `source-url` | `sourceId`, `url`, `title?` | A URL source cited by the model. |
| `source-document` | `sourceId`, `mediaType`, `title`, `filename?` | A document source cited by the model. |
| `reasoning-delta` | `text` | A chunk of the model's internal reasoning (chain-of-thought). |
| `step-start` | _(none)_ | Marks the beginning of an agent step (one LLM call in the tool loop). |
| `step-finish` | `usage`, `finishReason` | Marks the end of an agent step. `usage` contains token counts. |
| `file` | `mediaType`, `url`, `filename?` | A file generated by the model. |
| `memory-context` | `facts`, `episodes` | Memory chunks recalled for this prompt. No `threadId`. |
| `subagent-start` | `threadId`, `toolCallId`, `subagentName` | A subagent thread has started. The UI uses this to set up a nested thread view. |
| `subagent-done` | `threadId` | A subagent thread has completed. |

## Gateway Events

Events emitted by the Gateway's pub/sub system for cross-channel streaming. These extend `StreamEvent` with a `conversationId` field, plus gateway-specific event types.

### GatewayEvent

```ts
type GatewayEvent =
  | { type: "user-message"; conversationId: string; channelName: string; content: string }
  | { type: "done"; conversationId: string }
  | { type: "cleared"; conversationId: string }
  | { type: "error"; conversationId: string; message: string }
  | (StreamEvent & { conversationId: string });
```

Every `StreamEvent` variant can also appear as a `GatewayEvent` with `conversationId` attached. The additional gateway-only event types are:

| Type | Key Fields | Description |
|---|---|---|
| `user-message` | `conversationId`, `channelName`, `content` | A user sent a message in a conversation. Emitted so other subscribers (e.g. the web UI watching via WebSocket) see the incoming message. |
| `done` | `conversationId` | The agent has finished responding to a message. |
| `cleared` | `conversationId` | A conversation's history was cleared. |
| `error` | `conversationId`, `message` | An error occurred while processing a message. |

## Store Interface

The storage abstraction for persisting conversation history. All backends (SQLite, in-memory, etc.) implement this interface.

### IMessageStore

```ts
interface IMessageStore {
  // --- Message Management ---

  addMessage(
    conversationId: string,
    message: Omit<UIMessage, "id">,
    meta?: MessageMeta
  ): Promise<string>;

  getHistory(conversationId: string): Promise<UIMessage[]>;

  clearHistory(conversationId: string): Promise<void>;

  // --- Streaming Persistence ---

  createMessage(
    conversationId: string,
    role: "user" | "assistant",
    meta?: MessageMeta
  ): Promise<string>;

  appendPart(messageId: string, part: PandoraMessagePart): Promise<void>;

  updateTextPart(messageId: string, text: string): Promise<void>;

  updateToolResult(
    messageId: string,
    toolCallId: string,
    result: unknown
  ): Promise<void>;

  finalizeMessage(messageId: string): Promise<void>;

  accumulateUsage(
    messageId: string,
    usage: { inputTokens?: number; outputTokens?: number; totalTokens?: number }
  ): Promise<void>;

  // --- Conversation Management ---

  listConversations(channelName?: string): Promise<ConversationInfo[]>;

  deleteConversation(conversationId: string): Promise<void>;

  close(): Promise<void>;

  // --- Subagent Thread Management ---

  createSubagentConversation(
    parentId: string,
    toolCallId: string,
    subagentName: string,
    meta?: MessageMeta
  ): Promise<string>;

  linkToolToThread(
    messageId: string,
    toolCallId: string,
    threadId: string
  ): Promise<void>;

  getChildThreads(conversationId: string): Promise<ConversationInfo[]>;
}
```

#### Message Management Methods

| Method | Returns | Description |
|---|---|---|
| `addMessage(conversationId, message, meta?)` | `Promise<string>` | Store a complete message. Returns the generated message ID. For non-streaming use cases where the full message is already available. |
| `getHistory(conversationId)` | `Promise<UIMessage[]>` | Retrieve the full conversation history as an ordered array of messages. |
| `clearHistory(conversationId)` | `Promise<void>` | Delete all messages in a conversation while keeping the conversation record. |

#### Streaming Persistence Methods

These methods support incremental message building during streaming. The typical flow is: `createMessage` -> `appendPart`/`updateTextPart` (repeated) -> `finalizeMessage`.

| Method | Returns | Description |
|---|---|---|
| `createMessage(conversationId, role, meta?)` | `Promise<string>` | Create an empty message shell before streaming begins. Returns the message ID. |
| `appendPart(messageId, part)` | `Promise<void>` | Append a message part (text, tool call, source, reasoning, etc.) to an existing message. |
| `updateTextPart(messageId, text)` | `Promise<void>` | Update the last text part with accumulated content. Called repeatedly during streaming as text deltas arrive. |
| `updateToolResult(messageId, toolCallId, result)` | `Promise<void>` | Update a tool call part with its result, transitioning its state from `input-available` to `output-available`. |
| `finalizeMessage(messageId)` | `Promise<void>` | Mark a message as complete. Transitions the text part state from `streaming` to `done`. |
| `accumulateUsage(messageId, usage)` | `Promise<void>` | Add token usage counts to a message. Called on each `step-finish` event to accumulate totals across steps. |

#### Conversation Management Methods

| Method | Returns | Description |
|---|---|---|
| `listConversations(channelName?)` | `Promise<ConversationInfo[]>` | List conversations ordered by most recently updated. Optionally filter by channel name. |
| `deleteConversation(conversationId)` | `Promise<void>` | Delete a conversation and all its messages permanently. |
| `close()` | `Promise<void>` | Gracefully shut down the store -- flush pending writes, release connections, clean up resources. |

#### Subagent Thread Methods

| Method | Returns | Description |
|---|---|---|
| `createSubagentConversation(parentId, toolCallId, subagentName, meta?)` | `Promise<string>` | Create a child conversation for a subagent execution, linked to the parent via tool call ID. Returns the new conversation ID. |
| `linkToolToThread(messageId, toolCallId, threadId)` | `Promise<void>` | Link a tool call part to the subagent thread it spawned. Adds a `threadId` field to the tool part. |
| `getChildThreads(conversationId)` | `Promise<ConversationInfo[]>` | Get all subagent threads that are children of a given conversation. |

### ConversationInfo

Summary information about a stored conversation, returned by `listConversations` and `getChildThreads`.

```ts
interface ConversationInfo {
  id: string;
  /** Channel that created this conversation */
  channelName: string | null;
  /** Unix epoch seconds */
  createdAt: number;
  /** Unix epoch seconds */
  updatedAt: number;
  /** First user message, truncated */
  preview: string;
  /** Total number of messages */
  messageCount: number;
  /** Conversation type: 'root' for top-level, 'subagent' for child threads */
  type?: "root" | "subagent";
  /** Parent conversation ID (for subagent threads) */
  parentConversationId?: string;
  /** Tool call ID that spawned this thread (for subagent threads) */
  parentToolCallId?: string;
  /** Subagent name (for subagent threads) */
  subagentName?: string;
}
```

| Field | Type | Description |
|---|---|---|
| `id` | `string` | Unique conversation identifier. |
| `channelName` | `string \| null` | The channel that created this conversation, or `null` if unknown. |
| `createdAt` | `number` | Creation timestamp (Unix epoch seconds). |
| `updatedAt` | `number` | Last update timestamp (Unix epoch seconds). |
| `preview` | `string` | Truncated first user message, used for display in conversation lists. |
| `messageCount` | `number` | Total number of messages in the conversation. |
| `type` | `"root" \| "subagent"` | `"root"` for top-level conversations, `"subagent"` for child threads. |
| `parentConversationId` | `string` | Parent conversation ID. Only present for subagent threads. |
| `parentToolCallId` | `string` | The tool call ID that spawned this thread. Only present for subagent threads. |
| `subagentName` | `string` | Name of the subagent. Only present for subagent threads. |

## Memory Types

Types for the memory system, which provides episodic (interaction history) and semantic (facts/preferences) memory.

### Episode

An episodic memory record representing a past interaction event or conversation summary.

```ts
interface Episode {
  id: string;
  /** Summary of the interaction (user message + truncated assistant response) */
  content: string;
  /** Source conversation ID */
  conversationId?: string;
  /** Channel where this occurred */
  channelName?: string;
  /** User involved */
  userId?: string;
  /** Unix timestamp (seconds) */
  timestamp: number;
  /** Importance score (0-1) */
  importance: number;
  /** Optional tags for filtering */
  tags: string[];
}
```

| Field | Type | Description |
|---|---|---|
| `id` | `string` | Unique episode identifier. |
| `content` | `string` | Summary of the interaction (user message + truncated assistant response). |
| `conversationId` | `string` | The conversation this episode was extracted from. |
| `channelName` | `string` | Channel where the interaction occurred. |
| `userId` | `string` | The user involved in the interaction. |
| `timestamp` | `number` | When the interaction occurred (Unix epoch seconds). |
| `importance` | `number` | Importance score from `0` to `1`. Higher values indicate more significant interactions. |
| `tags` | `string[]` | Tags for categorical filtering. |

### Fact

A semantic memory record representing a piece of knowledge, user preference, or instruction.

```ts
interface Fact {
  id: string;
  /** The fact/preference/knowledge content */
  content: string;
  /** Category: user_preference, knowledge, instruction */
  category: string;
  /** Unix timestamp (seconds) */
  createdAt: number;
  /** Unix timestamp (seconds) */
  updatedAt: number;
  /** Source of this fact (e.g., conversation ID) */
  source?: string;
  /** Confidence score (0-1) */
  confidence: number;
}
```

| Field | Type | Description |
|---|---|---|
| `id` | `string` | Unique fact identifier. |
| `content` | `string` | The fact, preference, or knowledge content. |
| `category` | `string` | Category such as `"user_preference"`, `"knowledge"`, or `"instruction"`. |
| `createdAt` | `number` | When this fact was first recorded (Unix epoch seconds). |
| `updatedAt` | `number` | When this fact was last updated (Unix epoch seconds). |
| `source` | `string` | Source of the fact (e.g. a conversation ID). |
| `confidence` | `number` | Confidence score from `0` to `1`. |

### MemoryChunk

A chunk from memory search results. Memory records are split into chunks for embedding, and search returns the most relevant chunks.

```ts
interface MemoryChunk {
  /** Chunk ID */
  chunkId: string;
  /** Chunk text content */
  content: string;
  /** Parent record ID */
  parentId: string;
  /** Parent record type */
  parentType: "episode" | "fact";
  /** Hybrid search score (0-1) */
  score: number;
  /** Chunk index within parent (0-based) */
  chunkIndex: number;
  /** Unix timestamp - episodes only */
  timestamp?: number;
  /** Conversation ID - episodes only */
  conversationId?: string;
  /** Category - facts only */
  category?: string;
}
```

| Field | Type | Description |
|---|---|---|
| `chunkId` | `string` | Unique chunk identifier. |
| `content` | `string` | The text content of this chunk. |
| `parentId` | `string` | ID of the parent `Episode` or `Fact` record. |
| `parentType` | `"episode" \| "fact"` | Whether this chunk belongs to an episode or a fact. |
| `score` | `number` | Hybrid search relevance score from `0` to `1`. |
| `chunkIndex` | `number` | Position of this chunk within the parent record (0-based). |
| `timestamp` | `number` | Unix timestamp. Present only for episode chunks. |
| `conversationId` | `string` | Conversation ID. Present only for episode chunks. |
| `category` | `string` | Fact category. Present only for fact chunks. |

### MemorySearchResult

Generic search result wrapper with a relevance score. Used for backward compatibility.

```ts
interface MemorySearchResult<T> {
  item: T;
  /** Cosine similarity score (0-1) */
  score: number;
}
```

### MemorySearchResults

Combined chunk search results returned by `IMemoryProvider.search()`.

```ts
interface MemorySearchResults {
  /** Matching chunks from episodic memory */
  episodes: MemoryChunk[];
  /** Matching chunks from semantic memory */
  facts: MemoryChunk[];
}
```

## Memory Interfaces

Interfaces for implementing memory providers. A memory provider can support episodic memory, semantic memory, or both.

### IMemoryProvider

The top-level memory provider interface. Exposes optional episodic and semantic capabilities, a unified search method, and resource cleanup.

```ts
interface IMemoryProvider {
  /** Episodic memory capability (null if not supported) */
  readonly episodic: IEpisodicMemory | null;
  /** Semantic memory capability (null if not supported) */
  readonly semantic: ISemanticMemory | null;

  /** Search both memory types and return combined results */
  search(
    query: string,
    opts?: { limit?: number; minScore?: number; excludeConversationId?: string }
  ): Promise<MemorySearchResults>;

  /** Close the provider (release resources) */
  close(): Promise<void>;
}
```

| Member | Type | Description |
|---|---|---|
| `episodic` | `IEpisodicMemory \| null` | Episodic memory interface, or `null` if the provider does not support it. |
| `semantic` | `ISemanticMemory \| null` | Semantic memory interface, or `null` if the provider does not support it. |
| `search(query, opts?)` | `Promise<MemorySearchResults>` | Search across both memory types simultaneously. Returns combined chunks. `excludeConversationId` filters out episodes from the specified conversation. |
| `close()` | `Promise<void>` | Release all resources held by the provider. |

### IEpisodicMemory

Interface for episodic memory -- automatic logging of interaction events and past conversation summaries.

```ts
interface IEpisodicMemory {
  /** Add an episode (gateway calls this automatically after each turn) */
  addEpisode(episode: Omit<Episode, "id">): Promise<string>;

  /** Search episodes by semantic similarity (returns matching chunks) */
  searchEpisodes(
    query: string,
    opts?: { limit?: number; minScore?: number; since?: number }
  ): Promise<MemoryChunk[]>;

  /** Get a specific episode by ID */
  getEpisode(id: string): Promise<Episode | null>;

  /** Delete a specific episode */
  deleteEpisode(id: string): Promise<void>;

  /** Delete all episodes for a conversation (called when conversation is deleted) */
  deleteEpisodesForConversation(conversationId: string): Promise<number>;
}
```

| Method | Returns | Description |
|---|---|---|
| `addEpisode(episode)` | `Promise<string>` | Store a new episode. The `id` field is generated automatically. Returns the episode ID. The gateway calls this after each conversation turn. |
| `searchEpisodes(query, opts?)` | `Promise<MemoryChunk[]>` | Search episodes by semantic similarity. Options: `limit` (max results), `minScore` (threshold 0-1), `since` (Unix timestamp filter). |
| `getEpisode(id)` | `Promise<Episode \| null>` | Retrieve a specific episode by its ID. Returns `null` if not found. |
| `deleteEpisode(id)` | `Promise<void>` | Delete a specific episode and its chunks. |
| `deleteEpisodesForConversation(conversationId)` | `Promise<number>` | Delete all episodes associated with a conversation. Returns the count of deleted episodes. Called when a conversation is deleted. |

### ISemanticMemory

Interface for semantic memory -- agent-controlled storage of facts, preferences, and knowledge.

```ts
interface ISemanticMemory {
  /** Create or update a fact (deduplicates by similarity) */
  upsertFact(fact: Omit<Fact, "id" | "createdAt" | "updatedAt">): Promise<string>;

  /** Search facts by semantic similarity (returns matching chunks) */
  searchFacts(
    query: string,
    opts?: { limit?: number; minScore?: number; category?: string }
  ): Promise<MemoryChunk[]>;

  /** Delete a specific fact */
  deleteFact(id: string): Promise<void>;

  /** Get a fact by ID */
  getFact(id: string): Promise<Fact | null>;
}
```

| Method | Returns | Description |
|---|---|---|
| `upsertFact(fact)` | `Promise<string>` | Create or update a fact. Deduplicates by semantic similarity -- if a sufficiently similar fact exists, it is updated instead of creating a duplicate. The `id`, `createdAt`, and `updatedAt` fields are managed automatically. Returns the fact ID. |
| `searchFacts(query, opts?)` | `Promise<MemoryChunk[]>` | Search facts by semantic similarity. Options: `limit` (max results), `minScore` (threshold 0-1), `category` (filter by fact category). |
| `deleteFact(id)` | `Promise<void>` | Delete a specific fact and its chunks. |
| `getFact(id)` | `Promise<Fact \| null>` | Retrieve a specific fact by its ID. Returns `null` if not found. |
