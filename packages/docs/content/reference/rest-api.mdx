# REST API

The web channel serves HTTP REST endpoints on the configured port (default 3000). All endpoints require authentication via `Authorization: Bearer TOKEN` header.

For streaming communication, see the [WebSocket Protocol](/reference/websocket-protocol).

## Endpoints

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/api/validate` | `GET` | Token validation |
| `/api/conversations` | `GET` | List all conversations |
| `/api/conversations/:id/history` | `GET` | Get conversation messages |
| `/api/conversations/:id/threads` | `GET` | Get subagent threads for a conversation |
| `/api/conversations/:id` | `DELETE` | Delete a conversation |
| `/api/message` | `POST` | Non-streaming message (request/response) |
| `/api/clear` | `POST` | Clear conversation history |

## `GET /api/validate`

Validate a bearer token.

**Response:**
```json
{ "valid": true }
```

On failure (401):
```json
{ "valid": false }
```

---

## `GET /api/conversations`

List all conversations across all channels, ordered by most recently updated.

**Response:**
```json
{
  "conversations": [
    {
      "id": "web-1234567890",
      "channelName": "web",
      "createdAt": 1700000000,
      "updatedAt": 1700001000,
      "preview": "Hello, how are you?",
      "messageCount": 4,
      "type": "root"
    }
  ]
}
```

---

## `GET /api/conversations/:id/history`

Get the full message history for a conversation. Messages are AI SDK `UIMessage` objects with parts.

**Response:**
```json
{
  "messages": [
    {
      "id": "msg-abc",
      "role": "user",
      "parts": [{ "type": "text", "text": "Hello", "state": "done" }]
    },
    {
      "id": "msg-def",
      "role": "assistant",
      "parts": [{ "type": "text", "text": "Hi! How can I help?", "state": "done" }]
    }
  ]
}
```

---

## `GET /api/conversations/:id/threads`

Get subagent thread conversations that are children of the specified conversation.

**Response:**
```json
{
  "threads": [
    {
      "id": "thread-xyz",
      "channelName": "web",
      "createdAt": 1700000500,
      "updatedAt": 1700000600,
      "preview": "Write a sorting function...",
      "messageCount": 2,
      "type": "subagent",
      "parentConversationId": "web-1234567890",
      "parentToolCallId": "call-abc",
      "subagentName": "coder"
    }
  ]
}
```

---

## `DELETE /api/conversations/:id`

Delete a conversation and all its messages. Also deletes associated episodic memories.

**Response:**
```json
{ "ok": true }
```

---

## `POST /api/message`

Send a message and receive a complete (non-streaming) response.

**Request body:**
```json
{
  "content": "What is the weather today?",
  "conversationId": "web-my-conversation"
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `content` | `string` | Yes | Message text |
| `conversationId` | `string` | No | Conversation ID (auto-generated if omitted) |

**Response:**
```json
{ "response": "I don't have access to weather data..." }
```

**Errors:**
- `400` -- Missing `content` field
- `500` -- Internal processing error

---

## `POST /api/clear`

Clear all messages in a conversation.

**Request body:**
```json
{
  "conversationId": "web-my-conversation"
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `conversationId` | `string` | No | Defaults to `"web-default"` |

**Response:**
```json
{ "ok": true }
```

---

## CORS

All HTTP endpoints include permissive CORS headers:

```
Access-Control-Allow-Origin: *
Access-Control-Allow-Methods: GET, POST, DELETE, OPTIONS
Access-Control-Allow-Headers: Content-Type, Authorization
```

Preflight `OPTIONS` requests return `204 No Content` with CORS headers.
