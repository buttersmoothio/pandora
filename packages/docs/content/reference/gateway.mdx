# Gateway API

The Gateway is the central router in Pandora. It connects channels, the agent, the store, and memory. All message processing flows through it.

Defined in `packages/core/src/gateway.ts`, exported from `@pandora/core`.

## Constructor

```typescript
new Gateway(store: IMessageStore, agent: Agent, memory?: IMemoryProvider | null)
```

| Parameter | Type | Description |
|-----------|------|-------------|
| `store` | `IMessageStore` | Message store for conversation history |
| `agent` | `Agent` | AI agent for generating responses |
| `memory` | `IMemoryProvider \| null` | Optional memory provider for auto-recall and auto-episode |

## Methods

### `handleMessage`

```typescript
handleMessage(message: Message, capabilities: ChannelCapabilities): Promise<string>
```

Non-streaming message processing. Stores the user message, loads history, calls the agent, stores the response, and returns the full text. Internally delegates to `handleMessageStream` and collects the result.

| Parameter | Type | Description |
|-----------|------|-------------|
| `message` | `Message` | Incoming message from a channel |
| `capabilities` | `ChannelCapabilities` | Channel capabilities passed to the agent |

**Returns:** The complete assistant response text.

---

### `handleMessageStream`

```typescript
handleMessageStream(
  message: Message,
  capabilities: ChannelCapabilities,
  onEvent?: (event: StreamEvent) => void
): AsyncGenerator<string, void>
```

Streaming message processing. Yields text deltas as they arrive. The optional `onEvent` callback receives [`StreamEvent`](/reference/streaming#streamevent) objects for tool calls, sources, reasoning, memory context, subagent lifecycle events, and more.

| Parameter | Type | Description |
|-----------|------|-------------|
| `message` | `Message` | Incoming message from a channel |
| `capabilities` | `ChannelCapabilities` | Channel capabilities passed to the agent |
| `onEvent` | `(event: StreamEvent) => void` | Optional callback for stream events |

**Returns:** An async generator yielding text delta strings.

**Behavior:**
- Acquires a per-conversation lock (5-minute timeout) to prevent message interleaving
- Establishes a [`RequestContext`](/reference/types#request-context) for the duration of the request (available to tools via `requestContext.getStore()`)
- Stores the user message and creates an assistant message shell before streaming begins
- Auto-recalls memory hints before the agent prompt (limit: 8, minScore: 0.50, top 3 facts + 2 episodes after deduplication, truncated to 120 chars with IDs for on-demand retrieval via `getMemory`)
- Persists all parts incrementally as they stream in (text, tool calls, reasoning, sources, files)
- Manages subagent thread setup and persistence via `SubagentContext`
- Auto-creates an episodic memory entry after the response completes (fire-and-forget)
- Emits [`GatewayEvent`](/reference/streaming#gatewayevent)s to all subscribers for cross-channel streaming

---

### `subscribe`

```typescript
subscribe(conversationId: string, listener: GatewayListener): () => void
```

Subscribe to real-time events for a specific conversation. Returns an unsubscribe function.

```typescript
type GatewayListener = (event: GatewayEvent) => void;
```

```typescript
const unsubscribe = gateway.subscribe("conv-123", (event) => {
  console.log(event.type, event);
});

// Later:
unsubscribe();
```

---

### `subscribeAll`

```typescript
subscribeAll(listener: GatewayListener): () => void
```

Subscribe to events for ALL conversations. Returns an unsubscribe function. Used by the web UI for sidebar updates (conversation activity indicators).

---

### `getActiveStreamState`

```typescript
getActiveStreamState(conversationId: string): ActiveStreamState | undefined
```

Get the current state of an active stream. Used by [late-joining subscribers](/reference/streaming#late-joining-subscribers) to catch up on an in-progress response. Returns `undefined` if no stream is active for the conversation.

---

### `clearConversation`

```typescript
clearConversation(conversationId: string): Promise<void>
```

Clear all messages in a conversation. Emits a `cleared` event to subscribers.

---

### `deleteConversation`

```typescript
deleteConversation(conversationId: string): Promise<void>
```

Delete a conversation and all its messages. Also deletes associated episodic memories if a memory provider is configured.

---

### `listConversations`

```typescript
listConversations(channelName?: string): Promise<ConversationInfo[]>
```

List all conversations, optionally filtered by channel name. Results are ordered by most recently updated. See [`ConversationInfo`](/reference/store#conversationinfo).

---

### `getConversationHistory`

```typescript
getConversationHistory(conversationId: string): Promise<UIMessage[]>
```

Get the full message history for a conversation. Returns AI SDK `UIMessage` objects with parts (text, tool calls, reasoning, sources, etc.).

---

### `getChildThreads`

```typescript
getChildThreads(conversationId: string): Promise<ConversationInfo[]>
```

Get subagent thread conversations that are children of a parent conversation. See [`ConversationInfo`](/reference/store#conversationinfo).

### `registerChannel`

```typescript
registerChannel(channel: Channel): void
```

Register a channel for push capability lookup. Call this for channels that support proactive push notifications (scheduled reminders).

| Parameter | Type | Description |
|-----------|------|-------------|
| `channel` | `Channel` | Channel instance to register |

---

### `getChannel`

```typescript
getChannel(name: string): Channel | undefined
```

Get a registered channel by name. Returns `undefined` if the channel is not registered.

---

### `getStore`

```typescript
getStore(): IMessageStore
```

Get the message store instance. Used by the scheduler for task recovery on startup.

---

### `handleScheduledTask`

```typescript
handleScheduledTask(taskId: string): Promise<void>
```

Handle a scheduled task trigger. Called by the scheduler's `onTrigger` callback when a task is due.

| Parameter | Type | Description |
|-----------|------|-------------|
| `taskId` | `string` | ID of the triggered task |

**Behavior:**
- Looks up task metadata from the store
- Updates task status to `running`
- Builds a synthetic message with task instructions
- Processes through normal message flow (agent decides whether to push)
- Updates task status to `completed` (one-time) or `pending` (recurring)
- Handles failures with retry logic up to `maxFailures`

---

### `pushToChannel`

```typescript
pushToChannel(channelName: string, userId: string, content: string): Promise<void>
```

Push a message to a user via a channel. Used by the `pushMessage` tool for scheduled task notifications.

| Parameter | Type | Description |
|-----------|------|-------------|
| `channelName` | `string` | Name of the channel to push through |
| `userId` | `string` | User ID in the channel's native format |
| `content` | `string` | Message content to send |

**Throws:**
- Error if channel is not registered
- Error if channel doesn't support push (`supportsPush: false`)
- Error if channel declares `supportsPush` but doesn't implement `push()`

---

### `getContextState`

```typescript
getContextState(conversationId: string): Promise<ContextState>
```

Get context state for a conversation (for UI display on load). Computes token usage, health, and costs from stored history.
