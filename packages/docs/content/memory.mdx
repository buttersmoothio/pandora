# Memory

Pandora's memory features are configured on the **Memory** page in the web UI. After changing memory settings, **restart Pandora** for the changes to take effect.

## Working Memory

Working memory is a persistent scratchpad that Pandora maintains across all conversations. It automatically tracks important facts, preferences, and context as you chat — so you don't have to repeat yourself across different threads.

Unlike semantic recall (which searches past messages), working memory is always present in every conversation. Think of it as the things Pandora actively keeps in mind about you.

### Enabling

Toggle working memory on the **Memory** page. Once enabled, Pandora will start building up context as you interact with it.

### Custom Template

By default, Pandora decides what to remember on its own. You can optionally provide a template — a short instruction describing what kind of information it should track. Leave it empty to use the default behavior.

### How It Persists

Working memory is shared across all conversations — whether you're chatting in the web UI, through a connected channel, or when scheduled tasks run. Information learned in one conversation is available in all others.

## Semantic Recall

When enabled, Pandora uses **semantic recall** to automatically bring up relevant context from past conversations. Instead of starting fresh every time, it searches its conversation history for messages related to what you're discussing and includes them as context.

### How It Works

Semantic recall uses vector embeddings — numerical representations of meaning — to find related messages across all your past threads. When you send a message, Pandora searches for the most similar past messages to include alongside the conversation.

This means Pandora can remember things you've told it before, reference earlier conversations, and build on past context without you having to repeat yourself.

### Enabling

Toggle semantic recall on the **Memory** page. You'll also need to select an embedding provider and model (e.g. `text-embedding-3-small` from OpenAI).

The embedding provider can be different from your chat model provider — you can chat with Anthropic but use OpenAI for embeddings, for example.

The default embedding model is `openai/text-embedding-3-small` but you can change it to any embedding model from a configured provider.

### Requirements

- At least one AI provider with embedding models must be configured (API key set)
- The chosen embedding provider and model must be available
- Storage uses the same database as everything else — no additional setup
