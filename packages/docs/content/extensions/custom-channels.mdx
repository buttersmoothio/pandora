# Custom Channels

Channels connect Pandora to messaging platforms. This guide walks you through creating one.

## Example: Discord channel

Let's build a Discord channel that lets you chat with Pandora from Discord.

<Steps>

### Install dependencies

```bash
bun add discord.js
```

### Create the file

Create `packages/pandora/src/channels/discord/index.ts`:

```typescript
import {
  defineChannel,
  Gateway,
  type Channel,
  type ChannelCapabilities,
  type Message,
} from "@pandora/core";
import { Client, GatewayIntentBits } from "discord.js";

// What config this channel needs
interface DiscordConfig {
  token: string;
  ownerId: string;
}

// What formatting Discord supports
const CAPABILITIES: ChannelCapabilities = {
  supportsImages: true,
  supportsFiles: true,
  supportsRichText: true,
  supportsButtons: false,
  supportsStreaming: false,
  maxMessageLength: 2000,
};

class DiscordChannel implements Channel {
  private client: Client;
  private config: DiscordConfig;
  private gateway: Gateway;

  constructor(config: DiscordConfig, gateway: Gateway) {
    this.config = config;
    this.gateway = gateway;
    this.client = new Client({
      intents: [
        GatewayIntentBits.Guilds,
        GatewayIntentBits.GuildMessages,
        GatewayIntentBits.MessageContent,
      ],
    });
  }

  async start(): Promise<void> {
    // Handle incoming messages
    this.client.on("messageCreate", async (msg) => {
      // Ignore bots
      if (msg.author.bot) return;

      // Only respond to owner
      if (msg.author.id !== this.config.ownerId) return;

      // Build message for gateway
      const message: Message = {
        id: msg.id,
        conversationId: `discord-${msg.channelId}`,
        userId: msg.author.id,
        text: msg.content,
        timestamp: msg.createdAt,
      };

      // Show typing indicator
      await msg.channel.sendTyping();

      // Process through gateway
      const response = await this.gateway.handleMessage(
        message,
        CAPABILITIES
      );

      // Send response
      await msg.reply(response);
    });

    // Connect to Discord
    await this.client.login(this.config.token);
    console.log("[Channel] [discord] Started");
  }

  async stop(): Promise<void> {
    this.client.destroy();
  }
}

// Register this channel
export default defineChannel({
  name: "discord",
  configKey: "discord",
  create: (config, gateway) =>
    new DiscordChannel(config as DiscordConfig, gateway),
});
```

### Add to config

```jsonc
"channels": {
  "discord": {
    "token": "your-discord-bot-token",
    "ownerId": "your-discord-user-id"
  }
}
```

### Done!

Restart Pandora. Message your Discord bot!

## How channels work

```
User sends message on platform
         |
Channel receives it, builds a Message object
         |
Channel calls handler(message, capabilities)     -- non-streaming
   or    streamingHandler(message, capabilities)  -- streaming
         |
Gateway: stores user message, loads history, runs agent
         |
Channel sends response back to user
```

---

## The Channel interface

Your channel class must implement:

```typescript
interface Channel {
  start(): Promise<void>;  // Connect and start listening
  stop(): Promise<void>;   // Disconnect gracefully
}
```

---

## Capabilities

Tell the AI what formatting this platform supports. This matches the `ChannelCapabilities`
interface from `@pandora/core`:

```typescript
readonly capabilities: ChannelCapabilities = {
  supportsImages: false,      // Can send/receive images
  supportsFiles: false,       // Can send/receive files
  supportsRichText: true,     // Supports rich text formatting (Markdown, HTML)
  supportsButtons: false,     // Supports interactive buttons/keyboards
  supportsStreaming: true,    // Can stream responses token-by-token
  maxMessageLength: -1,       // Character limit per message (-1 = unlimited)
};
```

The AI uses this to format responses appropriately. For example, if `supportsRichText`
is `false`, the AI avoids Markdown formatting. If `maxMessageLength` is set, the
AI keeps responses within that limit.

## Message format

Convert platform messages to Pandora's format:

```typescript
const message: Message = {
  id: msg.id,                              // Unique message ID
  conversationId: `discord-${channelId}`,  // Unique per conversation
  userId: msg.author.id,                   // User identifier
  text: msg.content,                       // Message text
  timestamp: msg.createdAt,                // When sent
  media: [],                               // Optional attachments
};
```

### Handling media

If the platform supports attachments:

```typescript
const message: Message = {
  // ... other fields
  media: [
    {
      type: "image",
      url: attachment.url,
      mimeType: "image/png",
    },
  ],
};
```

## Streaming vs non-streaming

The gateway provides two handler types. Channels get them from the gateway at construction time.

### Non-streaming (e.g. Telegram)

Use `gateway.getHandler()` which returns a function that processes the full message and returns
the complete response text:

```typescript
class TelegramChannel implements Channel {
  private messageHandler: MessageHandler;

  constructor(config: TelegramConfig, gateway: Gateway) {
    this.messageHandler = gateway.getHandler();
  }

  // In your message listener:
  private async handleMessage(ctx: Context, text: string) {
    const message: Message = { /* ... */ };

    // Returns the full response when done
    const response = await this.messageHandler(message, this.capabilities);

    await ctx.reply(response);
  }
}
```

### Streaming (e.g. Web/WebSocket)

Use `gateway.getStreamingHandlerWithEvents()` which returns an async generator that yields
text deltas and accepts an `onEvent` callback for tool calls and other stream events:

```typescript
class WebChannel implements Channel {
  private streamingHandlerWithEvents: (
    message: Message,
    capabilities: ChannelCapabilities,
    onEvent?: (event: StreamEvent) => void
  ) => AsyncGenerator<string, void>;

  constructor(config: WebConfig, gateway: Gateway) {
    this.streamingHandlerWithEvents = gateway.getStreamingHandlerWithEvents();
  }

  // In your WebSocket handler:
  private async handleStream(ws: WebSocket, message: Message) {
    const stream = this.streamingHandlerWithEvents(
      message,
      this.capabilities,
      (event) => {
        // Forward stream events (tool-call, tool-result, etc.) to client
        ws.send(JSON.stringify(event));
      }
    );

    for await (const delta of stream) {
      ws.send(JSON.stringify({ type: "delta", text: delta }));
    }

    ws.send(JSON.stringify({ type: "done" }));
  }
}
```

There is also `gateway.getStreamingHandler()` which yields text deltas without the
`onEvent` callback, for channels that want streaming text but do not need tool call events.

```typescript
import { App } from "@slack/bolt";

class SlackChannel implements Channel {
  private app: App;
  private gateway: Gateway;

  constructor(config: SlackConfig, gateway: Gateway) {
    this.gateway = gateway;
    this.app = new App({
      token: config.botToken,
      signingSecret: config.signingSecret,
    });
  }

  async start(): Promise<void> {
    this.app.message(async ({ message, say }) => {
      if (message.user !== this.config.ownerId) return;

      const msg: Message = {
        id: message.ts,
        conversationId: `slack-${message.channel}`,
        userId: message.user,
        text: message.text,
        timestamp: new Date(parseFloat(message.ts) * 1000),
      };

      const response = await this.gateway.handleMessage(msg, CAPABILITIES);
      await say(response);
    });

    await this.app.start();
  }

  async stop(): Promise<void> {
    await this.app.stop();
  }
}
```

```typescript
import * as readline from "readline";

class CLIChannel implements Channel {
  private gateway: Gateway;
  private rl: readline.Interface;

  constructor(config: {}, gateway: Gateway) {
    this.gateway = gateway;
  }

  async start(): Promise<void> {
    this.rl = readline.createInterface({
      input: process.stdin,
      output: process.stdout,
    });

    console.log("Pandora CLI - Type your message:");

    this.rl.on("line", async (input) => {
      const message: Message = {
        id: Date.now().toString(),
        conversationId: "cli",
        userId: "cli-user",
        text: input,
        timestamp: new Date(),
      };

      const response = await this.gateway.handleMessage(
        message,
        {
          supportsImages: false,
          supportsFiles: false,
          supportsRichText: true,
          supportsButtons: false,
          supportsStreaming: false,
          maxMessageLength: -1,
        }
      );

      console.log(`\nPandora: ${response}\n`);
    });
  }

  async stop(): Promise<void> {
    this.rl.close();
  }
}
```

## Tips

- **Owner check** — Restrict to your user ID to prevent unauthorized use
- **Error handling** — Catch errors and send friendly messages
- **Typing indicators** — Show the user something is happening
- **Long messages** — Split responses that exceed platform limits
- **Graceful shutdown** — Clean up connections in `stop()`
