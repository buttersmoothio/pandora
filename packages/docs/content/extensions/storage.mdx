# Custom Storage

Storage backends persist conversation history. This guide walks you through creating one.

## Example: PostgreSQL backend

Let's build a Postgres storage backend.

### Step 1: Install dependencies

```bash
bun add pg
```

### Step 2: Create the file

Create `packages/pandora/src/store/postgres.ts`:

```typescript
import { defineStore, type IMessageStore, type ChatMessage } from "@pandora/core";
import { Pool } from "pg";

interface PostgresConfig {
  type: string;
  connectionString: string;
}

class PostgresStore implements IMessageStore {
  private pool: Pool;

  constructor(connectionString: string) {
    this.pool = new Pool({ connectionString });
  }

  async addMessage(conversationId: string, message: ChatMessage): Promise<void> {
    await this.pool.query(
      `INSERT INTO messages (conversation_id, role, content, created_at)
       VALUES ($1, $2, $3, NOW())`,
      [conversationId, message.role, JSON.stringify(message.content)]
    );
  }

  async getHistory(conversationId: string): Promise<ChatMessage[]> {
    const result = await this.pool.query(
      `SELECT role, content FROM messages
       WHERE conversation_id = $1
       ORDER BY created_at`,
      [conversationId]
    );

    return result.rows.map((row) => ({
      role: row.role,
      content: JSON.parse(row.content),
    }));
  }

  async clearHistory(conversationId: string): Promise<void> {
    await this.pool.query(
      "DELETE FROM messages WHERE conversation_id = $1",
      [conversationId]
    );
  }

  async close(): Promise<void> {
    await this.pool.end();
  }
}

export default defineStore({
  type: "postgres",
  create: (config) =>
    new PostgresStore((config as PostgresConfig).connectionString),
});
```

### Step 3: Create the table

Run this SQL on your Postgres database:

```sql
CREATE TABLE messages (
  id SERIAL PRIMARY KEY,
  conversation_id TEXT NOT NULL,
  role TEXT NOT NULL,
  content TEXT NOT NULL,
  created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_messages_conversation ON messages(conversation_id);
```

### Step 4: Add to config

```jsonc
"storage": {
  "type": "postgres",
  "connectionString": "postgresql://user:pass@localhost:5432/pandora"
}
```

### Step 5: Done!

Restart Pandora. Conversations are now stored in Postgres.

## The IMessageStore interface

Your storage backend must implement the full `IMessageStore` interface. It covers basic
message operations, streaming persistence, conversation management, and subagent threads:

```typescript
interface IMessageStore {
  // === Message Management ===

  /** Add a complete message (non-streaming, or already complete). Returns message ID. */
  addMessage(
    conversationId: string,
    message: Omit<UIMessage, "id">,
    meta?: MessageMeta
  ): Promise<string>;

  /** Get full conversation history */
  getHistory(conversationId: string): Promise<UIMessage[]>;

  /** Clear all messages in a conversation */
  clearHistory(conversationId: string): Promise<void>;

  // === Streaming Persistence ===

  /** Create a message shell before streaming begins. Returns message ID. */
  createMessage(
    conversationId: string,
    role: "user" | "assistant",
    meta?: MessageMeta
  ): Promise<string>;

  /** Append a part to an existing message (text, tool-call, etc.) */
  appendPart(messageId: string, part: PandoraMessagePart): Promise<void>;

  /** Update the last text part with accumulated text (during streaming) */
  updateTextPart(messageId: string, text: string): Promise<void>;

  /** Update a tool part with its result */
  updateToolResult(
    messageId: string,
    toolCallId: string,
    result: unknown
  ): Promise<void>;

  /** Finalize a message (mark text part as done) */
  finalizeMessage(messageId: string): Promise<void>;

  /** Accumulate token usage for a message (called on each step-finish) */
  accumulateUsage(
    messageId: string,
    usage: { inputTokens?: number; outputTokens?: number; totalTokens?: number }
  ): Promise<void>;

  // === Conversation Management ===

  /** List conversations, optionally filtered by channel. Ordered by most recently updated. */
  listConversations(channelName?: string): Promise<ConversationInfo[]>;

  /** Delete a conversation and all its messages */
  deleteConversation(conversationId: string): Promise<void>;

  /** Gracefully close the store (flush writes, release connections) */
  close(): Promise<void>;

  // === Subagent Thread Management ===

  /** Create a child conversation for a subagent execution. Returns conversation ID. */
  createSubagentConversation(
    parentId: string,
    toolCallId: string,
    subagentName: string,
    meta?: MessageMeta
  ): Promise<string>;

  /** Link a tool call to its spawned thread (updates tool part with threadId) */
  linkToolToThread(
    messageId: string,
    toolCallId: string,
    threadId: string
  ): Promise<void>;

  /** Get child threads (subagent conversations) for a parent conversation */
  getChildThreads(conversationId: string): Promise<ConversationInfo[]>;
}
```

### Method groups

| Group | Methods | Purpose |
|-------|---------|---------|
| **Basic** | `addMessage`, `getHistory`, `clearHistory` | Standard message CRUD |
| **Streaming** | `createMessage`, `appendPart`, `updateTextPart`, `updateToolResult`, `finalizeMessage`, `accumulateUsage` | Incremental persistence as tokens stream in |
| **Conversations** | `listConversations`, `deleteConversation`, `close` | Conversation lifecycle |
| **Subagent threads** | `createSubagentConversation`, `linkToolToThread`, `getChildThreads` | Child conversations for delegated agents |

The streaming methods are the core of how Pandora persists messages. The gateway calls
`createMessage()` to create a shell, then `appendPart()` and `updateTextPart()` as tokens
arrive, and finally `finalizeMessage()` when the stream is complete.

## Message format

Pandora uses the AI SDK's `UIMessage` type (aliased as `PandoraMessage`). Messages are parts-based,
supporting text, tool calls, reasoning, sources, and more:

```typescript
// UIMessage from AI SDK
interface UIMessage {
  id: string;
  role: "user" | "assistant" | "system";
  parts: MessagePart[];  // Text, tool calls, reasoning, sources, files, etc.
  // ... other fields
}
```

Store message parts as JSON to handle all part types:

```typescript
// Storing
await db.insert({
  parts: JSON.stringify(message.parts),
});

// Reading
return {
  id: row.id,
  role: row.role,
  parts: JSON.parse(row.parts),
};
```

## More examples

### Redis backend

```typescript
import { createClient } from "redis";

class RedisStore implements IMessageStore {
  private client: ReturnType<typeof createClient>;

  constructor(url: string) {
    this.client = createClient({ url });
  }

  async addMessage(conversationId: string, message: ChatMessage): Promise<void> {
    await this.client.connect();
    await this.client.rPush(
      `messages:${conversationId}`,
      JSON.stringify(message)
    );
  }

  async getHistory(conversationId: string): Promise<ChatMessage[]> {
    await this.client.connect();
    const messages = await this.client.lRange(`messages:${conversationId}`, 0, -1);
    return messages.map((m) => JSON.parse(m));
  }

  async clearHistory(conversationId: string): Promise<void> {
    await this.client.connect();
    await this.client.del(`messages:${conversationId}`);
  }

  async close(): Promise<void> {
    await this.client.quit();
  }
}

export default defineStore({
  type: "redis",
  create: (config) => new RedisStore((config as { url: string }).url),
});
```

Config:

```jsonc
"storage": {
  "type": "redis",
  "url": "redis://localhost:6379"
}
```

### MongoDB backend

```typescript
import { MongoClient, Collection } from "mongodb";

class MongoStore implements IMessageStore {
  private client: MongoClient;
  private collection: Collection;

  constructor(uri: string, dbName: string) {
    this.client = new MongoClient(uri);
    this.collection = this.client.db(dbName).collection("messages");
  }

  async addMessage(conversationId: string, message: ChatMessage): Promise<void> {
    await this.collection.insertOne({
      conversationId,
      ...message,
      createdAt: new Date(),
    });
  }

  async getHistory(conversationId: string): Promise<ChatMessage[]> {
    const docs = await this.collection
      .find({ conversationId })
      .sort({ createdAt: 1 })
      .toArray();

    return docs.map((doc) => ({
      role: doc.role,
      content: doc.content,
    }));
  }

  async clearHistory(conversationId: string): Promise<void> {
    await this.collection.deleteMany({ conversationId });
  }

  async close(): Promise<void> {
    await this.client.close();
  }
}

export default defineStore({
  type: "mongodb",
  create: (config) => {
    const { uri, database } = config as { uri: string; database: string };
    return new MongoStore(uri, database);
  },
});
```

Config:

```jsonc
"storage": {
  "type": "mongodb",
  "uri": "mongodb://localhost:27017",
  "database": "pandora"
}
```

## Tips

- **Index by conversation_id** — Queries filter by this field
- **Order by timestamp** — History should be chronological
- **Handle JSON content** — Content can be string or array
- **Connection pooling** — Reuse connections for performance
- **Graceful cleanup** — Close connections properly in `close()`
