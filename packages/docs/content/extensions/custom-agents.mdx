# Custom Agents

Subagents are task-specific agents that the operator can delegate to. This guide walks you through creating one.

## Example: Translator agent

Let's build a translator that handles language translation tasks.

<Steps>

### Create the file

Create `packages/pandora/src/subagents/translator.ts`:

```typescript
import { z } from "zod";
import { defineSubagent } from "@pandora/core";

export default defineSubagent({
  name: "translator",
  configKey: "translator",

  instructions: `You are an expert translator. Translate text between languages
accurately while preserving tone, idioms, and cultural context.

When translating:
1. Identify the source language if not specified
2. Translate to the requested target language
3. Note any idioms or cultural references that don't translate directly
4. Maintain the original meaning and tone`,

  toolDescription: "Translate text between languages with cultural awareness",

  inputSchema: z.object({
    text: z.string().describe("The text to translate"),
    targetLanguage: z.string().describe("The language to translate to"),
  }),

  inputField: "text",
});
```

### Add to config

Edit `config.jsonc`:

```jsonc
"agents": {
  "operator": { "model": "google/gemini-3-flash" },
  "translator": { "model": "anthropic/claude-sonnet-4.5" }
}
```

### Done!

Restart Pandora. Now you can ask "Translate 'Hello, how are you?' to Japanese."

The operator will recognize this as a translation task and delegate to your translator agent.

</Steps>

## How it works

```typescript
export default defineSubagent({
  name: "translator",        // Unique identifier for logging
  configKey: "translator",   // Key in config.ai.agents

  instructions: `...`,       // System prompt for this agent
  toolDescription: "...",    // What the operator sees when delegating

  inputSchema: z.object({    // What input this agent accepts
    text: z.string(),
    targetLanguage: z.string(),
  }),

  inputField: "text",        // Which field has the main content

  // Optional fields:
  getTools: (config) => ({}), // Override which tools this agent receives
  useMemory: false,           // Opt out of memory tools (default: true)
});
```

The operator sees your agent as a tool it can call. When the user asks something that matches your `toolDescription`, the operator delegates.

| Field | Required | Description |
|-------|----------|-------------|
| `name` | Yes | Unique identifier, used in logging |
| `configKey` | Yes | Key in `config.ai.agents` for model/temperature settings |
| `instructions` | Yes | System prompt for this agent |
| `toolDescription` | Yes | Controls when the operator decides to delegate to this agent |
| `inputSchema` | Yes | Zod schema defining what input the operator passes |
| `inputField` | No | Which schema field contains the main prompt (defaults to first field) |
| `getTools` | No | Override tool set; can be async. Omit for all configured tools |
| `useMemory` | No | Set to `false` to remove memory tools (default: `true`) |

## Input schema

The `inputSchema` defines what the operator passes to your agent:

```typescript
inputSchema: z.object({
  text: z.string().describe("The text to translate"),
  targetLanguage: z.string().describe("Target language"),
}),
```

The `.describe()` calls help the operator understand what to provide.

### inputField

The `inputField` tells Pandora which schema field contains the main prompt:

```typescript
inputField: "text",  // The 'text' field is the primary input
```

If not specified, uses the first field.

## Writing good instructions

Instructions are the system prompt for your agent. Be specific:

**Good:**
```typescript
instructions: `You are a code reviewer. Analyze code for:
- Bugs and potential issues
- Security vulnerabilities
- Performance problems
- Style and readability

Be specific and actionable in your feedback. Include line numbers.`,
```

**Not as good:**
```typescript
instructions: `You review code.`,
```

## Giving agents tools

By default, subagents have access to all configured tools. Use `getTools` to customize.

### No tools (for search-enabled models)

```typescript
export default defineSubagent({
  name: "webSearchNative",
  // ...

  getTools: () => ({}),  // No tools - model has built-in search
});
```

### Specific tools only

```typescript
import { createToolsForAgent } from "@pandora/core";

export default defineSubagent({
  name: "coder",
  // ...

  getTools: (config) => {
    // Only get tools available to 'coder'
    return createToolsForAgent("coder", config.tools ?? {});
  },
});
```

### Async getTools (dynamic tool loading)

`getTools` can be async -- this is why `Agent.create()` is an async factory rather than
a constructor. A real-world example is the `webSearchTool` subagent, which dynamically
loads the configured search backend:

```typescript
import { z } from "zod";
import type { Tool } from "ai";
import {
  defineSubagent,
  type AIConfig,
  getSearchTool,
  validateSearchBackend,
  getAvailableSearchBackends,
} from "@pandora/core";

function getConfiguredSearchTool(config: AIConfig): Record<string, Tool> {
  const agentConfig = config.agents.webSearchTool as
    | { model: string; searchBackend?: string }
    | undefined;

  const searchBackend = agentConfig?.searchBackend;
  if (!searchBackend) return {};

  const validationError = validateSearchBackend(searchBackend, config.tools ?? {});
  if (validationError) return {};

  const tool = getSearchTool(searchBackend, config.tools ?? {});
  return tool ? { [searchBackend]: tool } : {};
}

export default defineSubagent({
  name: "webSearchTool",
  configKey: "webSearchTool",

  instructions: "You are a web search specialist. Use the search tool to find current information.",

  toolDescription:
    "Search the web using an external search API (Tavily, Exa, Perplexity Search). " +
    "Works with any model. Best for: detailed research, multiple searches, flexible backend choice.",

  inputSchema: z.object({
    query: z.string().describe("The search query or research question"),
  }),

  inputField: "query",

  // Dynamically select the configured search backend
  getTools: (config: AIConfig) => getConfiguredSearchTool(config),

  // Search agents should not store/recall memories
  useMemory: false,
});
```

## useMemory

By default, all subagents have access to the memory tools (`remember`, `recall`, `getMemory`, `forget`).
Set `useMemory: false` to remove them:

```typescript
export default defineSubagent({
  name: "webSearchTool",
  // ...
  useMemory: false,  // No memory tools for this subagent
});
```

This is useful for subagents that only fetch external data and should not store or recall memories.

## Tips

- **Focused purpose** — Each agent should do one thing well
- **Clear instructions** — Be specific about what the agent should do
- **Good tool description** — Help the operator know when to delegate
- **Use the right model** — Powerful model for complex tasks, fast model for simple ones
- **Test the delegation** — Ask questions that should trigger your agent
