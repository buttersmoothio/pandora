# Extend Standard Tools

Standard library tools ship as separate `@pandora/tools-*` packages. They run directly in Node.js (host mode) and have full runtime access.

## Building a Tool

<Steps>

### Define the tool

Use `defineTool()` to create a tool. Here's a weather tool that fetches data from an external API:

```ts
import { z } from 'zod'
import { defineTool } from '@pandora/core/tools'

const weatherTool = defineTool({
  id: 'weather',
  description: 'Get current weather for a city',
  inputSchema: z.object({
    city: z.string().describe('City name'),
  }),
  annotations: {
    readOnlyHint: true,
    destructiveHint: false,
    idempotentHint: true,
  },
  execute: async (input) => {
    const key = process.env.OPENWEATHER_API_KEY
    const res = await fetch(
      `https://api.openweathermap.org/data/2.5/weather?q=${input.city}&appid=${key}`
    )
    return res.json()
  },
})
```

- **`annotations`** are MCP-compatible hints that help the UI decide whether to auto-approve or prompt for confirmation. A tool marked `readOnlyHint: true` and `destructiveHint: false` can be auto-approved.

### Package it

Each package exports a `createTools` factory that returns a tool record:

```ts
// @pandora/tools-datetime/src/index.ts
import type { ToolPackageFactory } from '@pandora/core/tools'
import { currentTime } from './current-time'

export const createTools: ToolPackageFactory = (_env) => ({
  'current-time': currentTime,
})
```

The factory receives environment variables so tools can read API keys or other configuration at load time.

A tool package follows this structure:

<FileTree>
  <FileTree.Folder name="packages/tools-example" defaultOpen>
    <FileTree.File name="package.json" />
    <FileTree.Folder name="src" defaultOpen>
      <FileTree.File name="index.ts" />

    <FileTree.File name="tsconfig.json" />
  </FileTree.Folder>
</FileTree>

The `package.json` declares `@pandora/core` as a peer dependency:

```json
{
  "name": "@pandora/tools-example",
  "type": "module",
  "exports": { ".": "./src/index.ts" },
  "peerDependencies": {
    "@pandora/core": "workspace:*"
  },
  "dependencies": {
    "zod": "^4.3.6"
  }
}
```

### Enable it

Core loads standard library packages dynamically and filters tools based on the user's configuration — only tools with `config.tools[id].enabled === true` are loaded. The default configuration enables the standard set.

</Steps>

## Manifests

Every `defineTool()` call registers a manifest in a global registry keyed by tool ID. This lets the UI display descriptions and approval prompts.

```ts
import { getAllManifests, getManifest, getManifests } from '@pandora/core/tools'

const manifest = getManifest(weatherTool)
const manifests = getManifests(toolRecord)
const all = getAllManifests() // every registered tool
```

## Reference

All types are exported from `@pandora/core/tools`:

```ts
import {
  defineTool,
  getAllManifests,
  getManifest,
  getManifests,
} from '@pandora/core/tools'

import type {
  ToolManifest,
  ToolAnnotations,
  ToolPackageFactory,
  ToolRecord,
} from '@pandora/core/tools'
```

### `DefineToolOptions`

| Field | Type | Required | Description |
|---|---|---|---|
| `id` | `string` | Yes | Unique tool identifier |
| `description` | `string` | Yes | What the tool does (shown to the LLM) |
| `inputSchema` | `z.ZodType` | Yes | Zod schema for input validation |
| `outputSchema` | `z.ZodType` | No | Optional Zod schema for output validation |
| `execute` | `(input, context) => Promise` | Yes | The tool's handler function |
| `requireApproval` | `boolean` | No | Require user approval before execution |
| `annotations` | `ToolAnnotations` | No | MCP-compatible UI hints |

### `ToolAnnotations`

```ts
interface ToolAnnotations {
  title?: string              // Display title in UI
  readOnlyHint?: boolean      // Safe to run without confirmation
  destructiveHint?: boolean   // May perform irreversible operations
  idempotentHint?: boolean    // Same args = no additional effect
}
```

### `ToolPackageFactory`

```ts
type ToolPackageFactory = (
  env: Record<string, string | undefined>,
) => ToolRecord
```

### `ToolManifest`

The manifest registered for each tool created via `defineTool()`. Retrieved via `getManifest(tool)`, `getManifests(toolRecord)`, or `getAllManifests()`.
