# SQLite Memory

File-based memory with vector search. The default and recommended option.

## Configuration

```jsonc
"memory": {
  "type": "sqlite",
  "path": "data/memory.db",
  "embeddingModel": "openai/text-embedding-3-small"
}
```

| Field | Default | Description |
|-------|---------|-------------|
| `type` | `sqlite` | Memory backend |
| `path` | `data/memory.db` | Database file location |
| `embeddingModel` | `openai/text-embedding-3-small` | Model for semantic search |

## Embedding models

Embedding models convert text into vectors for semantic search. Supported models:

| Model | Dimensions | Best for |
|-------|------------|----------|
| `openai/text-embedding-3-small` | 1536 | Fast, good quality (default) |
| `openai/text-embedding-3-large` | 3072 | Higher quality, slower |

The gateway API key is used for embedding requests. Embeddings are batch-generated for efficiency.

## Full example

```jsonc
{
  "$schema": "./config.schema.jsonc",
  "ai": {
    "gateway": { "apiKey": "your-gateway-key" },
    "agents": {
      "operator": { "model": "google/gemini-3-flash" }
    }
  },
  "storage": {
    "type": "sqlite",
    "path": "data/pandora.db"
  },
  "memory": {
    "type": "sqlite",
    "path": "data/memory.db",

  },
  "channels": {
    "telegram": {
      "token": "bot-token",
      "ownerId": "user-id"
    }
  }
}
```

## Managing memory

### Backup

```bash
cp data/memory.db data/memory.db.backup
```

### Clear all memory

```bash
rm data/memory.db
```

### View memory stats

```bash
sqlite3 data/memory.db "SELECT COUNT(*) FROM memory_episodes; SELECT COUNT(*) FROM memory_facts;"
```

For database schema, chunking strategy, hybrid search algorithm, fact deduplication, and other internals, see [Memory Reference — Internals](/reference/memory-reference#memory-internals).
