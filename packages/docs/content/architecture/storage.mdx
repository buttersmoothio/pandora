# Storage

Every storage provider produces two independent layers that share a database connection:

```
StorageResult
├── mastra   ← AI memory, threads, workflows (Mastra adapter)
└── config   ← Agent configuration (ConfigStore)
```

Providers are discovered at runtime via convention — `STORAGE_PROVIDER=postgres` imports `@pandora/storage-postgres`.

SQL-backed providers use a single dialect-aware class (`SQLConfigStore`) that handles SQLite, PostgreSQL, and MSSQL differences through a caller-provided `execute(sql, params)` function, keeping the class free of driver-specific imports.

In server mode, both storage and config are cached for the process lifetime. In serverless mode, fresh instances are created per request.
