# Inbox

The **Inbox** is a unified view of all [scheduled task](/user-guide/scheduling) results. Every task result passes through the inbox, even when it's also delivered to a channel like Telegram.

Each message has a subject, body, destination, and a delivery status — **Sent**, **Pending**, or **Failed**. Messages destined for the Web Inbox are immediately **Sent**. Messages routed to a channel start as **Pending** and update to **Sent** or **Failed** after delivery.

## Reading Messages

The inbox uses a two-pane layout: message list on the left, full message on the right. Selecting a message marks it as read automatically. Message bodies render as markdown.

## Archiving

Messages can be archived to keep the inbox clean. Toggle between the inbox and archive views with the button at the top. Archived messages can be restored at any time.

## Replying

If a message came from a scheduled task, a **Reply** button links to the original conversation thread so you can continue the discussion in chat.

## Deleting

Messages can be permanently deleted with a confirmation prompt.
