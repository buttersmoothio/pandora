# SQLite

SQLite is the default vector provider. It works out of the box with zero configuration, storing vectors in the same local database as everything else.

## Installation

The SQLite vector provider is included as a default dependency — no extra install needed.

If you removed it and want to add it back:

```bash
bun add @pandorakit/vector-libsql
```

## Configuration

| Variable | Required | Default | Description |
|---|---|---|---|
| `VECTOR_PROVIDER` | No | `libsql` | Set to `libsql` or omit entirely |
| `DATABASE_URL` | No | `file:data/pandora.db` | SQLite / LibSQL connection URL |
| `DATABASE_AUTH_TOKEN` | No | — | Auth token for remote Turso databases |

### Local File Mode

By default, vectors are stored in the same SQLite database at `data/pandora.db` alongside your other data. No env vars are needed.

```bash
# No env vars needed — just start the server
bun run dev
```

You can customize the file path:

```bash
DATABASE_URL=file:/path/to/my.db
```

### Remote Mode (Turso)

To use a hosted [Turso](https://turso.tech) database:

```bash
DATABASE_URL=libsql://your-db-name-org.turso.io
DATABASE_AUTH_TOKEN=your-auth-token
```
