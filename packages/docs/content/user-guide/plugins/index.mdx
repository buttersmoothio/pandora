# Plugins

Plugins extend what Pandora can do — tools for checking the time or searching the web, agents that specialize in research or coding, and channels that connect to messaging platforms like Telegram. Manage them all from the **Plugins** page in the web UI.

## Filtering

Use the filter pills at the top of the page to narrow by capability:

- **All** — every installed plugin
- **Tools** — plugins that provide tools
- **Agents** — plugins that provide specialist agents
- **Channels** — plugins that connect to messaging platforms

## Enabling a Plugin

To activate a plugin:

1. Set the plugin's required environment variables (listed in its settings dialog)
2. Open the plugin's settings dialog and fill in any required configuration fields, then click **Save**
3. Toggle the plugin on

A plugin appears under **Disabled** until all requirements are met. Plugins with only optional configuration load with defaults automatically.

## Permissions

Tool plugins display badges for the capabilities they use: **Time**, **Network**, **Env**, **Filesystem**, and **Random**. These tell you at a glance what a plugin can access.

## Require Approval

You can enable **Require Approval** on individual tools within a plugin. Open the plugin's settings dialog to see the tool list with per-tool approval toggles.

When a tool with this setting is triggered:

- **Web UI** — the tool card opens automatically showing the parameters. Click **Approve** to let it run or **Deny** to block it. The conversation resumes after your choice.
- **Telegram** — an inline keyboard appears with **Approve** and **Deny** buttons below the tool name and parameters.

## Agent Model Overrides

For plugins that provide agents, the settings dialog shows each agent with an option to set a custom model (provider + model name). If left blank, the agent uses the default operator model.
