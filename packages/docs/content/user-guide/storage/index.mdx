# Storage

Pandora uses SQLite by default — a local database is created automatically at `data/pandora.db` with no configuration needed. This is the recommended setup for most users.

## Using a Different Database

If you're deploying to a serverless environment or want to use existing infrastructure, Pandora supports other databases. Set `STORAGE_PROVIDER` and the required connection variables, then install the matching package.

For example, to use PostgreSQL:

```bash
# .env
STORAGE_PROVIDER=postgres
DATABASE_URL=postgresql://user:pass@localhost:5432/pandora
```

```bash
bun add @pandora/storage-postgres
```

Restart the server after switching. Schema tables are created automatically on startup.

<Callout type="warning">
  DynamoDB requires manual table creation before use. See the [DynamoDB page](/user-guide/storage/dynamodb) for setup instructions.
</Callout>
