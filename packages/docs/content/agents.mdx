# Agents

Pandora uses a multi-agent system where a main **operator** can delegate tasks to specialized **sub-agents**.

## How it works

```
User Message → Operator → Response
                  ↓
            (decides to delegate)
                  ↓
              Sub-agent → Back to Operator → Response
```

The **operator** is your main AI assistant. It handles most conversations directly, but when it recognizes a specialized task, it can delegate to a sub-agent:

- Coding question? → Delegates to `coder`
- Need research? → Delegates to `research`
- Need current info? → Delegates to `webSearchNative`

Each sub-agent has focused instructions and can use different AI models.

## Configuration

### Operator only (minimal)

```jsonc
"agents": {
  "operator": { "model": "anthropic/claude-sonnet-4.5" }
}
```

The operator alone is quite capable for general use.

### With specialists

```jsonc
"agents": {
  "operator": { "model": "anthropic/claude-sonnet-4.5" },
  "coder": { "model": "anthropic/claude-sonnet-4.5" },
  "research": { "model": "openai/gpt-4o" },
  "webSearchNative": { "model": "perplexity/sonar-pro" }
}
```

You can enable any combination — just include the ones you want.

## Built-in Agents

### Operator (required)

The main AI that handles all conversations.

| Field | Value |
|-------|-------|
| Config key | `operator` |
| Purpose | General conversation, task routing |
| Recommended models | `anthropic/claude-sonnet-4.5`, `openai/gpt-4o` |

### Coder

Programming specialist for code questions.

| Field | Value |
|-------|-------|
| Config key | `coder` |
| Purpose | Writing, debugging, reviewing code |
| Recommended models | `anthropic/claude-sonnet-4.5`, `anthropic/claude-opus-4` |

**Triggers on:** "Write a function...", "Debug this code...", "Review this PR..."

### Research

Information gathering and explanations.

| Field | Value |
|-------|-------|
| Config key | `research` |
| Purpose | Answering questions, explaining concepts |
| Recommended models | `openai/gpt-4o`, `anthropic/claude-sonnet-4.5` |

**Triggers on:** "Explain how...", "What is...", "Compare X and Y..."

### Web Search (Native)

Live internet searches using search-enabled models.

| Field | Value |
|-------|-------|
| Config key | `webSearchNative` |
| Purpose | Current information, news, real-time data |
| Supported models | `perplexity/sonar-pro`, `perplexity/sonar`, `google/gemini-2.0-flash` |

**Triggers on:** "What happened today...", "Current price of...", "Latest news about..."

This is the simplest way to add web search — the model handles everything internally.

### Web Search (Tool-based)

Web search using external APIs (Tavily, Exa, Perplexity).

| Field | Value |
|-------|-------|
| Config key | `webSearchTool` |
| Purpose | Web search with any model |
| Requires | A search tool configured (see [Tools](/tools)) |

```jsonc
"tools": {
  "tavilySearch": { "apiKey": "your-key" }
},
"agents": {
  "webSearchTool": {
    "model": "anthropic/claude-sonnet-4.5",
    "searchBackend": "tavilySearch"
  }
}
```

Use this when you want a specific model to process search results.

## Choosing models

| Strategy | Example |
|----------|---------|
| Same everywhere | Use `claude-sonnet-4.5` for all agents |
| Task-optimized | Best coding model for coder, fast model for research |
| Cost-optimized | Expensive model for operator, cheaper for simple tasks |

**Example: Cost-optimized**

```jsonc
"agents": {
  "operator": { "model": "anthropic/claude-sonnet-4.5" },
  "coder": { "model": "anthropic/claude-sonnet-4.5" },
  "research": { "model": "openai/gpt-4o-mini" }
}
```

---

## Do I need sub-agents?

**Start without them** if you're just getting started. The operator handles most things well.

**Add sub-agents** when you frequently ask certain types of questions:
- Lots of coding → add `coder`
- Need explanations → add `research`
- Need current info → add `webSearchNative`
