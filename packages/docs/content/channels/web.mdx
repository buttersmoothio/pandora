# Web

HTTP + WebSocket API for browser-based chat. Includes a Next.js web UI.

## Setup

<Steps>

### Configure Pandora

```jsonc
"channels": {
  "web": {
    "port": 3000,
    "token": "pick-a-secret-token"
  }
}
```

### Start everything

```bash
bun run dev
```

### Open the web UI

Go to `http://localhost:3001` and enter your token.

</Steps>

## Features

- **Streaming responses** — See text as it's generated
- **Conversation sidebar** — All conversations from all channels
- **Tool visibility** — See what tools the AI uses
- **Chain-of-thought** — View AI reasoning (when supported)
- **Cross-channel sync** — Watch Telegram chats update live
- **Subagent threads** — See delegated tasks running in parallel
- **Memory context** — View recalled memories for each prompt
- **Source citations** — URL and document sources inline

## Authentication

**HTTP requests:** Include `Authorization: Bearer YOUR_TOKEN` in every request header.

**WebSocket:** Pass the token as a query parameter when connecting:

```
ws://localhost:3000/ws?token=YOUR_TOKEN
```

Requests with missing or invalid tokens receive a `401 Unauthorized` response.

## API and Protocol

The web channel exposes HTTP REST endpoints and a WebSocket interface on the configured port (default 3000). For full details see:

- [REST API Reference](/reference/rest-api) — CRUD for conversations, non-streaming message endpoint, token validation
- [WebSocket Protocol](/reference/websocket-protocol) — Streaming chat, cross-channel watching, conversation subscriptions

---

## Cross-Channel Streaming

The web UI can watch **any** conversation, including from other channels like Telegram. When you open a conversation in the sidebar, the UI subscribes to its live events — you see text, tool calls, and subagent threads in real time regardless of which channel originated the conversation.

See the [WebSocket Protocol — Cross-Channel](/reference/websocket-protocol#cross-channel) section for details.
