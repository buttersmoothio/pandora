# Telegram

Chat with Pandora from your phone via a Telegram bot. Owner-only, supports text, photos, documents, voice, audio, and video.

## Setup

<Steps>

### Create a bot

Message [@BotFather](https://t.me/BotFather), send `/newbot`, follow the prompts. Save the token.

### Get your user ID

Message [@userinfobot](https://t.me/userinfobot). Save your ID.

### Configure Pandora

```jsonc
"channels": {
  "telegram": {
    "token": "123456789:ABCdefGHIjklMNOpqrsTUVwxyz",
    "ownerId": "123456789"
  }
}
```

### Start and chat

```bash
bun run dev
```

Message your bot on Telegram!

</Steps>

## Capabilities

| Capability | Value |
|------------|-------|
| `supportsImages` | `true` |
| `supportsFiles` | `true` |
| `supportsRichText` | `true` — Telegram HTML formatting |
| `supportsButtons` | `true` — inline keyboards |
| `supportsStreaming` | `false` — would require message editing |
| `maxMessageLength` | `4096` |

---

## Supported Input Types

The bot handles six message types:

| Type | Description |
|------|-------------|
| **Text** | Plain text content |
| **Photo** | Image attachment (largest resolution selected) |
| **Document** | File attachment with metadata |
| **Voice** | Voice message in Ogg format |
| **Audio** | Audio file with metadata |
| **Video** | Video file with metadata |

Captions are used as the message text when present; otherwise a placeholder like `[Photo received]` or `[Voice message: 12s]` is used.

---

## Security

<Callout type="warning">
The bot only responds to the configured `ownerId`. Messages from any other user are silently ignored (logged but not replied to). The `/start` command is the one exception -- non-owners receive a brief "this bot is private" reply.
</Callout>

---

## Commands

| Command | Effect |
|---------|--------|
| `/start` | Start a new conversation (generates a fresh conversation ID) |

---

## Conversation Management

- On first message (or after a restart), the bot resumes the **most recent conversation** from the store. This provides cross-channel continuity -- a conversation started on the web UI can be continued in Telegram.
- `/start` creates a brand-new conversation ID, discarding the previous one.
- Conversations are visible from all channels. A Telegram conversation appears in the web UI sidebar and can be watched live.

---

## Response Formatting

Responses support rich formatting — markdown is automatically converted to Telegram-compatible HTML (bold, italic, code blocks, links, blockquotes, lists, etc.). If formatting fails, the response falls back to plain text.

Long responses are automatically split into multiple messages to stay within Telegram's character limit.

The bot shows a typing indicator while generating responses.
