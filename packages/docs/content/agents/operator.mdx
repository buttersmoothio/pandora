# Operator

The operator is the core agent at the heart of Pandora. Every message from every channel flows through it — it handles general conversation, coordinates tools, and routes tasks to subagents.

## Configuration

The operator is the only **required** agent. It is configured at `ai.agents.operator` in `config.jsonc`:

```jsonc
{
  "ai": {
    "gateway": {
      "apiKey": "your-ai-gateway-api-key"
    },
    "agents": {
      "operator": {
        "model": "google/gemini-3-flash",
        "temperature": 0,
        "maxSteps": 20
      }
    }
  }
}
```

### Config options

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `model` | `string` | *required* | Gateway model ID (e.g. `"google/gemini-3-flash"`, `"anthropic/claude-sonnet-4.5"`) |
| `temperature` | `number` | `0` | Controls randomness. `0` = almost deterministic, higher values = more random. |
| `maxOutputTokens` | `number` | — | Maximum number of tokens to generate |
| `maxSteps` | `number` | `20` | Maximum tool loop iterations before stopping |

## How it works

The operator builds its context dynamically on every request, including channel capabilities, available tools, subagent descriptions, and any recalled memories. It then streams a response, calling tools or delegating to subagents as needed.

- **Tools** — The operator has access to all configured action tools plus memory tools (when memory is enabled).
- **Delegation** — Each configured subagent appears as a tool the operator can call. The operator decides when to delegate based on the subagent's description, and always receives the subagent's response before replying to the user.
- **Streaming** — Responses stream in real time on channels that support it (like the web UI). Non-streaming channels (like Telegram) collect the full response before sending.

For a detailed walkthrough of how the operator processes messages, delegates to subagents, and manages streaming, see [Architecture](/reference/architecture).

## Model selection

The operator handles every message, so model cost and latency directly affect responsiveness across all channels. Use a fast model with good tool-calling support. See [Choosing models](/agents#choosing-models) for recommendations.
