# Agents

Pandora uses a multi-agent system where a main **operator** can delegate tasks to **subagents**.

## How it works

```
User Message → Operator → Response
                  ↓
            (decides to delegate)
                  ↓
              Subagent → Back to Operator → Response
```

The **operator** is your main AI assistant. It handles most conversations directly, but when it recognizes a specialized task, it can delegate to a subagent:

- Coding question? → Delegates to `coder`
- Need research? → Delegates to `research`
- Need current info? → Delegates to `webSearchNative`

Each subagent has its own instructions and can use a different AI model.

## Configuration

<Tabs items={['Recommended', 'With all subagents']}>
<Tabs.Tab>

```jsonc
"agents": {
  "operator": { "model": "google/gemini-3-flash" },
  "webSearchNative": { "model": "perplexity/sonar-pro" }
}
```

We recommend always enabling `webSearchNative` — it gives the operator access to live internet search, which is useful for a wide range of questions. Without it, the operator can only answer from its training data.

</Tabs.Tab>
<Tabs.Tab>

```jsonc
"agents": {
  "operator": { "model": "google/gemini-3-flash" },
  "coder": { "model": "anthropic/claude-sonnet-4.5" },
  "research": { "model": "openai/gpt-5.2" },
  "webSearchNative": { "model": "perplexity/sonar-pro" }
}
```

You can enable any combination — just include the ones you want.

</Tabs.Tab>

## Choosing models

| Strategy | Example |
|----------|---------|
| Cost-optimized | Fast model for operator, capable models for subagents |
| Task-optimized | Best coding model for coder, search model for research |
| Same everywhere | Use one model for simplicity |

**Recommended setup:**

```jsonc
"agents": {
  "operator": { "model": "google/gemini-3-flash" },
  "webSearchNative": { "model": "perplexity/sonar-pro" },
  "coder": { "model": "anthropic/claude-sonnet-4.5" },
  "research": { "model": "openai/gpt-5.2" }
}
```

The operator handles routing and simple tasks — use a fast, cheap model with good tool-calling. Always include `webSearchNative` for live internet access. Other subagents are optional and do the heavy lifting for their domains — use capable models there.

## Do I need subagents?

**Always enable `webSearchNative`** — it gives the operator live internet access, which is valuable for almost any use case.

**Add more subagents** when you frequently ask certain types of questions:
- Lots of coding → add `coder`
- Need explanations → add `research`
