# Web Search Agents

Live internet searches for current information. Pandora offers two approaches: **native search** (model-integrated) and **tool-based search** (external API).

All search agents share the same base config options as other agents (`temperature`, `maxOutputTokens`, `maxSteps`). See [Configuration Reference](/reference/configuration-reference#aiagents) for the full table.

## Native Search (`webSearchNative`)

Uses models with built-in search capabilities. The agent detects the provider from the model prefix and configures the appropriate search integration automatically.

```jsonc
"agents": {
  "webSearchNative": { "model": "perplexity/sonar-pro" }
}
```

### Examples

Perplexity Sonar (simplest -- no extra SDK needed):

```jsonc
"agents": {
  "webSearchNative": { "model": "perplexity/sonar-pro" }
}
```

Google Gemini with grounding (requires `@ai-sdk/google`):

```jsonc
"agents": {
  "webSearchNative": { "model": "google/gemini-3-flash" }
}
```

OpenAI search model (requires `@ai-sdk/openai`):

```jsonc
"agents": {
  "webSearchNative": { "model": "openai/gpt-4o-mini-search-preview" }
}
```

### When to Use

- You want the simplest setup with minimal configuration
- You are already using a search-enabled model (Perplexity Sonar, Gemini, OpenAI search)
- You want faster responses (single model call, search integrated into generation)
- You want to avoid additional API costs beyond the model itself

---

## Tool-Based Search (`webSearchTool`)

Uses an external search API as a tool, which works with **any model**. Configure a search backend under `ai.tools`, then reference it in the agent config.

```jsonc
"tools": {
  "tavilySearch": { "apiKey": "tvly-..." }
},
"agents": {
  "webSearchTool": {
    "model": "google/gemini-3-flash",
    "searchBackend": "tavilySearch"
  }
}
```

### Available Backends

| Backend | Config key | Description |
|---------|------------|-------------|
| Tavily | `tavilySearch` | General web search |
| Exa | `exaSearch` | AI-powered search with content extraction |
| Perplexity Search | `perplexitySearch` | Perplexity Search API (distinct from Sonar models) |

See [Web Search Tools](/tools/web-search) for backend setup details and API key links.

### When to Use

- You want to use a model that does not have native search (e.g., Claude, Llama)
- You want more control over the search backend and its behavior
- You need a specific search provider for your use case (e.g., Exa for semantic/research queries)
- You want to combine search with other tools in the same agent

---

## Comparison

| | Native Search | Tool-Based Search |
|--|---------------|-------------------|
| Agent config key | `webSearchNative` | `webSearchTool` |
| Model requirement | Search-enabled (Perplexity, OpenAI search, Gemini) | Any model |
| Additional API keys | None (may need provider SDK installed) | Search API key required |
| Speed | Faster (single integrated call) | Slower (model calls search tool, then processes results) |
| Search control | Limited to provider defaults | Choice of backend, configurable per-tool |

## Triggers

Both agents are triggered by prompts like:

- "What happened today..."
- "Current price of..."
- "Latest news about..."
- "Search for..."
