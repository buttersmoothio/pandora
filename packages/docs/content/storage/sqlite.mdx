# SQLite Storage

File-based database. The default and recommended option.

## Configuration

```jsonc
"storage": {
  "type": "sqlite",
  "path": "data/pandora.db"
}
```

| Field | Default | Description |
|-------|---------|-------------|
| `type` | `sqlite` | Storage backend |
| `path` | `data/pandora.db` | Database file location |

## Features

- Conversations persist across restarts
- WAL mode for concurrent reads during streaming
- Foreign key constraints with CASCADE DELETE
- Single file, easy to backup

## Backup

```bash
cp data/pandora.db data/pandora.db.backup
```

## Clear History

```bash
# Clear a specific conversation (messages and parts cascade-delete)
sqlite3 data/pandora.db "DELETE FROM conversations WHERE id = 'your-id';"
```

<Callout type="warning">
To clear **everything**, delete the database file: `rm data/pandora.db`. Pandora will recreate it on next start.
</Callout>

For database schema details, streaming persistence internals, and subagent thread storage, see [Architecture — Storage Internals](/reference/architecture#storage-internals).
