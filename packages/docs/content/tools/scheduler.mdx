# Scheduler Tools

Schedule reminders, recurring tasks, and push notifications. Automatically enabled when the scheduler is configured.

## Configuration

Enable the scheduler in your config:

```jsonc
"scheduler": {
  "type": "simple",
  "pollInterval": 10000
}
```

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `type` | `string` | No | `"simple"` | Scheduler backend. Currently only `"simple"` is available. |
| `pollInterval` | `number` | No | `10000` | How often to check for due tasks (milliseconds). |

See [Configuration Reference](/reference/configuration-reference#scheduler) for all options.

## Tools

When the scheduler is enabled, these tools become available to the operator:

### scheduleReminder

Schedule a one-time reminder.

```
"Remind me in 2 hours to check the deployment"
"Set a reminder for tomorrow at 9am to call John"
"Remind me next Monday at 3pm about the meeting"
```

**Parameters:**

| Field | Type | Description |
|-------|------|-------------|
| `when` | `string` | Natural language time (e.g., "in 30 minutes", "tomorrow at 9am") |
| `description` | `string` | What to remind about |
| `context` | `string` | Optional additional context |

**Time formats supported:**
- Relative: "in 30 minutes", "in 2 hours", "in 3 days"
- Absolute: "tomorrow at 9am", "next Friday at 2pm", "March 15th at noon"
- Named: "next Monday", "this weekend"

### scheduleRecurring

Schedule a recurring task.

```
"Every day at 9am, send me a standup reminder"
"Every Monday at 2pm, remind me about the team meeting"
"Every hour, check if there are new messages"
```

**Parameters:**

| Field | Type | Description |
|-------|------|-------------|
| `schedule` | `string` | Natural language schedule or cron expression |
| `description` | `string` | What this recurring task is for |
| `maxRuns` | `number` | Optional limit on number of runs |

**Schedule formats supported:**
- Daily: "every day at 9am", "daily", "every morning"
- Weekly: "every Monday", "every weekday", "every weekend"
- Interval: "every hour", "every 30 minutes"
- Cron: `"0 9 * * *"` (raw cron expressions)

### listScheduled

List all scheduled tasks for the current conversation.

```
"What reminders do I have?"
"Show my scheduled tasks"
"List upcoming reminders"
```

Returns pending reminders and recurring tasks with their status.

### cancelScheduled

Cancel a scheduled task by ID.

```
"Cancel that reminder"
"Stop the daily standup reminder"
```

**Parameters:**

| Field | Type | Description |
|-------|------|-------------|
| `taskId` | `string` | The ID of the task to cancel |

### pushMessage

Send a proactive message to the user. Used automatically when scheduled tasks trigger.

**Parameters:**

| Field | Type | Description |
|-------|------|-------------|
| `message` | `string` | The message content to send |

This tool is only available on channels that support push notifications (Telegram, Web with active connection).

## Example conversation

```
User: Remind me in 30 minutes to take a break

Assistant: I have set a reminder for you. In 30 minutes, I will notify you to take a break.

[30 minutes later...]

Assistant: Hey! Time to take a break. Stand up, stretch, and rest your eyes for a few minutes.
```

## How it works

1. When you ask to schedule something, the agent parses your request and creates a task
2. The task is stored in the database with all metadata
3. The scheduler polls for due tasks and triggers them
4. When a task triggers, the agent processes it and may send you a push notification
5. For recurring tasks, the next run is automatically scheduled

Tasks survive server restarts - pending tasks are recovered from the database on startup.
