# Dynamic Tool Resolution

Some tools can't be defined statically — they depend on which API keys are available or user configuration. For these cases, your entry point can export an async `resolveTools` function that runs at load time and returns `Tool` objects dynamically.

This is the mechanism behind the search plugins (e.g. `@pandorakit/tavily-search`, `@pandorakit/brave-search`): each checks whether its API key is available and returns the appropriate tool.

```ts
// src/index.ts
import type { ResolveToolsContext, ResolveToolsResult } from '@pandorakit/sdk/tools'
import { resolveSearchTool } from './resolve'

export const tools = []

export async function resolveTools(ctx: ResolveToolsContext): Promise<ResolveToolsResult> {
  const preferred = ctx.pluginConfig?.searchBackend as string | undefined
  const result = resolveSearchTool({ preferred, env: ctx.env })
  return { tools: result.tool ? [result.tool] : [], alerts: result.alerts }
}
```

A plugin can have both a static `tools` array and a `resolveTools` function. All tools — static and dynamically resolved — are loaded when the plugin is active.

The `resolveTools` function receives a `ResolveToolsContext`:

```ts
interface ResolveToolsContext {
  pluginConfig: PluginConfig  // validated plugin config
  env: Record<string, string | undefined>
}
```

Return a `ResolveToolsResult` — an object with `tools` (array of `Tool`) and optional `alerts`:

```ts
interface ResolveToolsResult {
  tools: Tool[]
  alerts?: Alert[]
}
```

## Alerts

`resolveTools` can return diagnostic alerts alongside tools:

```ts
export async function resolveTools({ env }: ResolveToolsContext): Promise<ResolveToolsResult> {
  if (!env.MY_API_KEY) {
    return {
      tools: [],
      alerts: [{ level: 'warning', message: 'API key not set' }],
    }
  }
  return {
    tools: [myTool],
    alerts: [{ level: 'info', message: 'Using MyService' }],
  }
}
```

Alerts are stored at load time and surfaced in the Plugins page and the `/api/plugins` endpoint. Levels are `'info'` (informational) and `'warning'` (actionable).
